Czym jest Emmet?	Wtyczka wbudowana w VS Code, która pomaga efektywniej pisać kod HTML i CSS dzięki systemowi skrótów.

Co oznacza skrót SVG?	Scalable Vector Graphics (Skalowalna Grafika Wektorowa).
Czym różni się grafika wektorowa od rastrowej?	Wektorowa opiera się na matematycznych wzorach, rastrowa na siatce pikseli.
Dlaczego SVG nie tracą jakości przy powiększaniu?	Ponieważ są zdefiniowane za pomocą formuł matematycznych, a nie ustalonej liczby pikseli.
Na jakim języku znaczników opiera się kod SVG?	Na XML.
Wymień typowe zastosowania formatu SVG.	Ikony, wykresy/grafy, proste ilustracje, tła z powtarzalnym wzorem.
W jakich sytuacjach NIE należy używać SVG?	Do zdjęć fotorealistycznych lub obrazów z bardzo dużą ilością detali i złożonych tekstur.
"Dlaczego pliki SVG są ""czytelne dla człowieka"" (human-readable)?"	Ponieważ są plikami tekstowymi opartymi na XML, a nie binarnymi (jak JPEG).
Za co odpowiada atrybut xmlns w SVG?	Określa przestrzeń nazw XML (XML NameSpace), niezbędną do poprawnego renderowania przez przeglądarkę.
Za co odpowiada atrybut viewBox w SVG?	Definiuje granice widoku, proporcje (aspect ratio) i punkt początkowy układu współrzędnych SVG.
Wymień przykładowe podstawowe elementy (tagi) budujące SVG.	&lt;rect&gt;, &lt;circle&gt;, &lt;path&gt;, &lt;text&gt;.
Na czym polega metoda &quot;Inline&quot; przy dodawaniu SVG?	Na wklejeniu kodu SVG bezpośrednio do pliku HTML, zamiast linkowania go jako zewnętrzny plik.
Jaka jest główna zaleta metody &quot;Inline&quot; dla SVG?	Umożliwia zmianę właściwości elementów SVG (np. fill, stroke) za pomocą CSS i JavaScript.
Jakie są wady metody &quot;Inline&quot; dla SVG?	Pogarsza czytelność kodu HTML, utrudnia cachowanie i może opóźnić ładowanie strony przy dużych plikach.
Jaka jest główna wada linkowania SVG jako zwykłego obrazka (<img> lub background-image)?	Brak dostępu do wewnętrznej struktury SVG (nie można stylować jego elementów przez CSS/JS).
Jaka jest główna wada linkowania SVG jako zwykłego obrazka (&lt;img&gt; lub background-image)?	Brak dostępu do wewnętrznej struktury SVG (nie można stylować jego elementów przez CSS/JS).
Jaki znacznik służy do stworzenia tabeli w HTML?	&lt;table&gt;
Z czego składają się tabele HTML?	Z wierszy i kolumn (struktura dwuwymiarowa).
Jaki znacznik definiuje wiersz w tabeli?	&lt;tr&gt; (table row)
Jaki znacznik definiuje komórkę nagłówkową w tabeli?	&lt;th&gt; (table header)
Jaki znacznik definiuje standardową komórkę z danymi w tabeli?	&lt;td&gt; (table data)
Dlaczego nie należy używać tabel do tworzenia układu strony (layoutu)?	Ponieważ jest to niesemantyczne (mylące dla czytników ekranowych), utrudnia responsywność (RWD) i tworzy skomplikowany kod.
Do czego służy znacznik &lt;caption&gt;?	Definiuje tytuł lub opis tabeli (powinien być pierwszym elementem wewnątrz &lt;table&gt;).
Do czego służy atrybut &quot;scope&quot; w nagłówkach tabeli (&lt;th&gt;)?	Pomaga czytnikom ekranowym zrozumieć, czy dany nagłówek dotyczy całego wiersza (row), czy kolumny (col).
Jaka składnia Emmet utworzy element &lt;p class=&quot;text&quot;&gt;&lt;/p&gt;?	p.text
Co to są &quot;default styles&quot; w kontekście przeglądarek?	Są to style domyślne aplikowane automatycznie przez przeglądarkę do każdej strony internetowej.
Skąd technicznie pochodzą domyślne style widoczne na stronie bez własnego CSS?	Z arkuszy stylów klienta użytkownika (user-agent stylesheets).
Jaki jest cel istnienia {{c1::user-agent stylesheets}}?	Zapewnienie podstawowego, czytelnego wyglądu strony internetowej, która nie posiada własnych stylów CSS.
Prawda czy Fałsz: Domyślne style (user-agent styles) są identyczne w każdej przeglądarce (Chrome, Firefox, Safari).	Fałsz (różnią się nieznacznie w zależności od przeglądarki).
Jaki jest typowy domyślny styl dla elementu &lt;h1&gt;?	Większa i pogrubiona czcionka.
Jaki jest typowy domyślny styl dla nieodwiedzonego elementu &lt;a&gt;?	Niebieski kolor tekstu i podkreślenie.
Wymień dwie właściwości CSS (properties), które są często domyślnie ustawiane przez przeglądarkę i mogą przeszkadzać w layoutcie.	Margin oraz padding.
W jaki sposób najprościej nadpisać domyślne style przeglądarki?	Napisać własną regułę CSS dla danego elementu w swoim arkuszu stylów.
Dlaczego reguły w Twoim arkuszu stylów nadpisują reguły przeglądarki?	Ponieważ style autora mają wyższy priorytet (precedence) niż style user-agent.
Co to jest CSS Reset?	Jest to arkusz stylów zawierający reguły mające na celu usunięcie lub zmianę domyślnych stylów user-agent.
Jaki jest główny cel używania CSS Reset w kontekście kompatybilności?	Zniwelowanie niespójności w wyglądzie elementów między różnymi przeglądarkami.
Użycie CSS Reset zapewnia deweloperowi tzw. {{c1::czystą kartę (clean slate)}} do nakładania własnych stylów bez zakłóceń.	czystą kartę (clean slate)
Prawda czy Fałsz: Używanie CSS Reset jest obowiązkowe w każdym projekcie.	Fałsz (jest to opcjonalne).
Co oznacza stwierdzenie, że CSS Reset jest &quot;subiektywny&quot; (opinionated)?	Oznacza to, że odzwierciedla on preferencje i wybory dewelopera, który go stworzył.
Czy deweloper musi korzystać z gotowych rozwiązań CSS Reset?	Nie, może stworzyć własny reset lub nie używać żadnego.
{{c1::User-agent stylesheet}} to inna nazwa na domyślne style wbudowane w przeglądarkę.	User-agent stylesheet
Dlaczego nagłówki (h1-h6) mają różne rozmiary, nawet gdy nie dodamy żadnego CSS?	Ponieważ przeglądarka nakłada na nie style z user-agent stylesheet.
Prawda czy Fałsz: Twoje style CSS mają niższy priorytet niż style user-agent.	Fałsz (mają wyższy priorytet).
Jaka jest alternatywa dla używania gotowego (prebuilt) CSS Reset?	Napisanie własnego resetu lub ręczne nadpisywanie stylów w miarę potrzeb.
Z czym najczęściej zmagają się początkujący, jeśli nie zresetują stylów przeglądarki (podaj przykład związany z odstępami)?	Z nieoczekiwanymi marginesami (margins) i dopełnieniem (padding) wokół elementów.
Jaka jest główna różnica między jednostkami absolutnymi a relatywnymi w CSS?	Jednostki absolutne są zawsze takie same, natomiast jednostki relatywne zmieniają się w zależności od kontekstu.
Dlaczego {{c1::px}} jest uznawany za jednostkę absolutną?	Ponieważ rozmiar piksela nie zmienia się w relacji do innych elementów na stronie.
Jaka jest jedyna jednostka absolutna zalecana do stosowania w projektach internetowych (web projects)?	Piksel (px).
W jakim kontekście mają sens jednostki takie jak `in` (cale) lub `cm` (centymetry) w CSS?	W kontekście druku (print setting).
Do czego odnosi się jednostka {{c1::em}} (gdy nie jest używana do określania font-size)?	Do rozmiaru czcionki (font-size) danego elementu.
Jeśli element ma `font-size: 16px`, a jego szerokość ustawisz na `4em`, ile wyniesie szerokość w pikselach?	64px (16 * 4).
Do czego odnosi się jednostka `em`, gdy jest używana do ustawienia właściwości `font-size`?	Do rozmiaru czcionki (font-size) elementu nadrzędnego (rodzica).
Do czego zawsze odnosi się jednostka {{c1::rem}}?	Do rozmiaru czcionki elementu głównego (root element: :root lub html).
Jaka jest główna zaleta stosowania `rem` nad `em`?	Unikamy złożoności związanej ze śledzeniem rozmiaru czcionki rodzica (brak efektu kaskadowego powiększania/zmniejszania).
Dlaczego zaleca się stosowanie {{c1::rem}} do definiowania rozmiarów czcionek na stronie?	Ponieważ szanuje to ustawienia przeglądarki użytkownika (zwiększa dostępność/readability).
Co oznacza skrót jednostki `vh`?	Viewport Height (wysokość obszaru widocznego).
Co oznacza skrót jednostki `vw`?	Viewport Width (szerokość obszaru widocznego).
Ile procent wysokości viewportu to `1vh`?	1%.
Prawda czy Fałsz: `1vw` jest równe 10% szerokości viewportu.	Fałsz (jest równe 1%).
Podaj przykład zastosowania dla jednostek viewportu (`vh`/`vw`).	Sekcje typu &quot;full-height hero&quot; lub interfejsy przypominające aplikacje pełnoekranowe.
Kiedy warto użyć jednostki `%` (procent) zamiast `vw` dla szerokości?	Gdy chcemy, aby szerokość zależała od elementu nadrzędnego (rodzica), a nie od szerokości całego okna przeglądarki.
Prawda czy Fałsz: Powinieneś nauczyć się na pamięć wszystkich dostępnych jednostek CSS przed rozpoczęciem pracy.	Fałsz (należy dobierać jednostki w zależności od pożądanego zachowania elementu).
Jeśli chcesz, aby margines skalował się wraz z rozmiarem czcionki głównej, jakiej jednostki użyjesz?	rem
W jaki sposób w dokumentacji MDN oznaczane są typy wartości (value types)?	Są otoczone nawiasami kątowymi, np. &lt;color&gt; lub &lt;length&gt;.
Co oznacza termin &quot;data type&quot; w kontekście CSS?	To inna nazwa na &quot;value type&quot; (typ wartości) – terminy te są używane zamiennie.
Jaki typ danych numerycznych w CSS reprezentuje liczbę całkowitą (np. 1024)?	{{c1::&lt;integer&gt;}}
Jaki typ danych numerycznych w CSS reprezentuje liczbę dziesiętną (np. 0.25)?	{{c1::&lt;number&gt;}}
Co to jest typ {{c1::&lt;dimension&gt;}} w CSS?	To liczba z dołączoną jednostką (np. 10px, 45deg, 5s).
Czy wartość procentowa (np. 50%) w CSS jest stała?	Nie, jest zawsze relatywna względem innej wartości (np. elementu rodzica).
Wymień jedną jednostkę długości absolutnej (absolute length unit), która jest powszechnie używana na ekranach.	px (piksel).
Ile fizycznych pikseli urządzenia to zawsze 1px w CSS?	To zależy (na ekranach HD 1px może obejmować wiele fizycznych pikseli
Ile pikseli (px) mieści się w jednym calu (in) według standardu CSS?	96px
Dlaczego jednostki takie jak `cm`, `mm` czy `in` są rzadko używane w projektowaniu stron na ekrany?	Ponieważ są bardziej przydatne w druku (print) i nie zawsze oddają fizyczne wymiary na różnych ekranach.
Do czego odnosi się jednostka `em`, gdy jest używana w właściwości `font-size`?	Do rozmiaru czcionki elementu rodzica (parent element).
Do czego odnosi się jednostka `em`, gdy jest używana w właściwościach innych niż `font-size` (np. width)?	Do rozmiaru czcionki tego samego elementu (my own font-size).
Do czego zawsze odnosi się jednostka `rem`?	Do rozmiaru czcionki elementu głównego (root element: &lt;html&gt;).
Co oznacza skrót `rem`?	Root em.
Co się stanie z rozmiarem elementów opartych na `rem`, jeśli zmienisz `font-size` w elemencie `html`?	Wszystkie te elementy przeskalują się relatywnie do nowej wartości bazowej.
Jaka jest różnica w dziedziczeniu rozmiaru między zagnieżdżonymi listami używającymi `em` a `rem`?	Zagnieżdżone `em` kumulują się (rosną/maleją wykładniczo), a `rem` pozostają stałe względem korzenia.
Jeśli ustawisz `width: 50%` dla elementu, względem czego zostanie ona obliczona?	Względem szerokości elementu rodzica (parent container).

W jakim zakresie mieści się wartość właściwości `opacity`?	Od 0 (pełna przezroczystość) do 1 (pełne krycie).

Jaka jest różnica między użyciem `opacity` a kanałem alfa w kolorze (np. w `rgb` lub `hex`)?	`opacity` czyni przezroczystym cały element i jego dzieci, a kanał alfa tylko konkretny kolor (np. tła).
Co oznacza litera &quot;H&quot; w modelu kolorów HSL/HWB i w jakich jednostkach jest wyrażana?	Hue (barwa), wyrażana jako kąt (0-360 stopni) na kole barw.

Czy słowa kluczowe w CSS (np. `red`, `solid`, `center`) są traktowane jako typ String?	Nie, są to identyfikatory (identifiers) i nie używa się przy nich cudzysłowu.
Kiedy w CSS używamy cudzysłowu dla wartości tekstowych?	Gdy wartość jest faktycznym ciągiem znaków (String), np. w właściwości `content`.

Prawda czy Fałsz: Można w pełni ostylować rozwijaną listę opcji (`&lt;option&gt;`) w standardowym elemencie `&lt;select&gt;` używając tylko CSS.	Fałsz (lista rozwijana jest renderowana przez system operacyjny).

Co trzeba zrobić, aby w pełni kontrolować wygląd kalendarza w formularzu?	Należy użyć biblioteki JavaScript (custom date picker) zamiast natywnego `&lt;input type=&quot;date&quot;&gt;`.
Jaka jest najważniejsza, &quot;złota zasada&quot; bezpieczeństwa aplikacji internetowych?	Nigdy nie ufaj danym pochodzącym z przeglądarki (użytkownika).
Co oznacza skrót XSS?	Cross-Site Scripting.
Na czym polega atak XSS (Cross-Site Scripting)?	Na wstrzyknięciu złośliwego skryptu (uruchamianego po stronie klienta) do przeglądarki innego użytkownika za pośrednictwem strony internetowej.
Czym różni się &quot;Reflected XSS&quot; od &quot;Persistent (Stored) XSS&quot;?	Reflected XSS zwraca złośliwy skrypt natychmiastowo i jednorazowo (np. w wyniku wyszukiwania), a Persistent XSS zapisuje go na serwerze (np. w komentarzu), atakując każdego, kto wyświetli daną treść.
Jaka jest najskuteczniejsza obrona przed atakami XSS?	Sanityzacja danych wejściowych (input sanitization) i wyłączanie/usuwanie znaczników HTML mogących uruchomić kod (np. `&lt;script&gt;`).
Na czym polega atak SQL Injection?	Na wstrzyknięciu fragmentu kodu SQL przez dane wejściowe użytkownika, co pozwala na nieautoryzowany dostęp lub modyfikację bazy danych.

Co oznacza skrót CSRF?	Cross-Site Request Forgery.
Na czym polega atak CSRF?	Atakujący zmusza przeglądarkę zalogowanego użytkownika do wykonania niechcianej akcji (np. przelewu) na zaufanej stronie bez wiedzy ofiary.
Dlaczego atak CSRF jest skuteczny, nawet jeśli atakujący nie zna hasła ofiary?	Ponieważ przeglądarka ofiary automatycznie dołącza pliki cookie (z sesją logowania) do każdego żądania wysyłanego do danej domeny, nawet jeśli żądanie zostało zainicjowane przez inną stronę.
Jaki jest powszechny mechanizm obrony przed CSRF?	Wymaganie, aby każde żądanie modyfikujące dane (POST) zawierało unikalny, generowany przez serwer &quot;token&quot; (tajny klucz), którego atakujący nie zna.
Na czym polega atak Clickjacking?	Na umieszczeniu niewidocznej warstwy (np. przezroczystego iframe) nad widoczną stroną, co sprawia, że użytkownik klika w coś innego, niż myśli.
Jakie nagłówki HTTP pomagają chronić przed Clickjackingiem?	Nagłówki zabraniające osadzania strony w ramkach (iframe) na innych witrynach (np. X-Frame-Options lub Content Security Policy).
Na czym polega atak DoS (Denial of Service)?	Na zalaniu serwera ogromną liczbą fałszywych żądań, aby uniemożliwić dostęp prawowitym użytkownikom.
Co to jest &quot;Directory Traversal&quot;?	Próba dostępu do plików poza dozwolonym katalogiem serwera poprzez użycie znaków nawigacji po ścieżkach (np. `../../`).
Dlaczego HTTPS jest kluczowy dla bezpieczeństwa formularzy logowania?	Ponieważ szyfruje dane przesyłane między klientem a serwerem, chroniąc hasła i tokeny sesji przed podsłuchaniem.
W jaki sposób tworzy się wyrażenie regularne za pomocą literału (literal syntax)?	Poprzez otoczenie wzorca ukośnikami, np. `const re = /ab+c/
W jaki sposób tworzy się wyrażenie regularne za pomocą konstruktora?	Używając `new RegExp()`, np. `const re = new RegExp(&quot;ab+c&quot;)
Kiedy należy używać literału wyrażenia regularnego (`/abc/`) zamiast konstruktora?	Gdy wzorzec jest stały (nie zmienia się). Zapewnia to kompilację podczas ładowania skryptu i lepszą wydajność.
Kiedy należy używać konstruktora `new RegExp()`?	Gdy wzorzec jest dynamiczny (np. pochodzi z danych wprowadzonych przez użytkownika) lub zmienia się w trakcie działania programu.
Jaki znak służy do &quot;uciekaczki&quot; (escaping) znaków specjalnych w Regex?	Backslash `\` (odwrócony ukośnik).
Jak należy zapisać wyrażenie `/a*b/` używając konstruktora `RegExp` (uwzględniając escaping w stringu)?	`new RegExp(&quot;a\\*b&quot;)` (należy podwoić backslash, ponieważ jest to literał łańcuchowy).
Co robi metoda `RegExp.prototype.test()`?	Sprawdza, czy wzorzec występuje w ciągu znaków. Zwraca `true` lub `false`.
Co robi metoda `RegExp.prototype.exec()`?	Wykonuje wyszukiwanie w ciągu znaków. Zwraca tablicę z wynikami (i grupami przechwytującymi) lub `null` w przypadku braku dopasowania. 
Wymień metody obiektu `String`, które obsługują wyrażenia regularne.	`match()`, `matchAll()`, `replace()`, `replaceAll()`, `search()`, `split()`
Co oznacza flaga `g` w wyrażeniu regularnym?	Global search (wyszukiwanie globalne - nie zatrzymuje się po pierwszym dopasowaniu).
Co oznacza flaga `i` w wyrażeniu regularnym?	Case-insensitive (ignorowanie wielkości liter).
Co oznacza flaga `m` w wyrażeniu regularnym?	Multiline (wielowierszowość). Sprawia, że `^` i `$` dopasowują się do początku/końca każdej linii, a nie tylko całego ciągu. 
Jaka jest różnica w działaniu `str.match(re)` a `re.exec(str)` gdy użyta jest flaga `g`?	`str.match(re)` zwraca tablicę wszystkich dopasowań naraz (bez szczegółów grup). `re.exec(str)` zwraca jedno dopasowanie na raz, aktualizując `lastIndex`, co pozwala na iterację w pętli.
Co zwraca metoda `str.search(re)`?	Indeks pierwszego dopasowania lub `-1`, jeśli nie znaleziono.
Do czego służą nawiasy `()` w wyrażeniu regularnym?	Tworzą grupę przechwytującą (capturing group), która zapamiętuje dopasowany podciąg do późniejszego wykorzystania.

Jaki kwantyfikator oznacza &quot;1 lub więcej wystąpień&quot;?	`+`
Jaki kwantyfikator oznacza &quot;0 lub 1 wystąpienie&quot; (opcjonalność)?	`?`
Jaki wynik da `const re = /a*/	` dla ciągu `&quot;bbbb&quot;`?

Jaki atrybut HTML sprawia, że pole formularza musi zostać wypełnione przed wysłaniem?	required
Dlaczego z punktu widzenia UX i dostępności samo dodanie atrybutu `required` nie wystarcza?	Należy wizualnie oznaczyć pole (np. gwiazdką `*` w etykiecie), aby użytkownik wiedział, że jest wymagane przed próbą wysłania.

Jaki atrybut pozwala walidować wartość pola na podstawie wyrażenia regularnego (Regex)?	{{c1::pattern}} 
Jaki jest domyślny komunikat przeglądarki, gdy dane nie pasują do atrybutu `pattern` i jak go ulepszyć?	Komunikat to zazwyczaj ogólne &quot;Proszę dopasować do wymaganego formatu&quot;. Należy użyć atrybutu `title` lub `placeholder`, aby pokazać użytkownikowi oczekiwany format (np. kod pocztowy).
Jakie dwie pseudoklasy CSS pozwalają stylować pola w zależności od wyniku walidacji (np. zielona ramka dla poprawnych, czerwona dla błędnych)?	:valid oraz :invalid 
Czy walidacja po stronie klienta (HTML/JS) jest wystarczająca do zapewnienia bezpieczeństwa danych?	Nie, zawsze niezbędna jest walidacja po stronie serwera (server-side validation), ponieważ walidację w przeglądarce można łatwo ominąć.
Dlaczego walidacja formularzy po stronie klienta (client-side) NIE może być traktowana jako jedyny środek bezpieczeństwa?	Ponieważ walidację w przeglądarce można łatwo ominąć (np. wyłączając JS lub modyfikując żądanie sieciowe)
Jaka jest główna zaleta walidacji po stronie klienta (w porównaniu do walidacji tylko na serwerze)?	Lepsze UX: użytkownik otrzymuje natychmiastową informację zwrotną bez konieczności oczekiwania na przeładowanie strony lub odpowiedź serwera.
Co robi atrybut `novalidate` dodany do elementu `&lt;form&gt;`?	Wyłącza domyślne dymki z błędami i automatyczne blokowanie wysyłania przez przeglądarkę, ale nadal pozwala korzystać z API walidacji w JavaScript i pseudoklas CSS.

Jaki atrybut ARIA warto dodać do elementu wyświetlającego komunikat błędu, aby czytniki ekranowe odczytały go natychmiast po pojawieniu się?	`aria-live=&quot;polite&quot;` (lub &quot;assertive&quot;).

Jakie jest zalecane rozwiązanie dla pola karty kredytowej, aby wymusić klawiaturę numeryczną na mobile, ale uniknąć spinnerów?	Użycie `&lt;input type=&quot;text&quot;&gt;` z atrybutem `inputmode=&quot;numeric&quot;`. 
Dlaczego nie należy polegać wyłącznie na atrybucie `placeholder` jako etykiecie pola?	Ponieważ znika on po wpisaniu znaku, co utrudnia użytkownikowi przypomnienie sobie celu pola, oraz często ma zbyt niski kontrast (dostępność). 
W obiekcie `ValidityState` (JS), która właściwość zwróci `true`, jeśli wartość jest mniejsza niż atrybut `min`?	{{c1::rangeUnderflow}}
W obiekcie `ValidityState` (JS), która właściwość zwróci `true`, jeśli wartość nie pasuje do typu pola (np. zły format emaila)?	{{c1::typeMismatch}}
Jaka pseudoklasa CSS pozwala na stylowanie elementu rodzica (np. kontenera `div`), gdy znajdujący się wewnątrz niego input posiada focus?	{{c1:: :focus-within}}
Jaka jest główna wada tworzenia własnych kontrolek formularza (np. custom date picker) zamiast używania natywnych?	Trudności z zapewnieniem pełnej dostępności (klawiatura, czytniki ekranu) i obsługi na wszystkich urządzeniach (dotyk, różne rozmiary ekranu). 
Jakie zdarzenie (event) jest wyzwalane na każdym nieprawidłowym polu formularza podczas próby wysłania (jeśli nie użyto `novalidate`)?	`invalid` (zdarzenie to nie bąbelkuje / does not bubble).
Gdzie jest najlepsze miejsce na wyświetlanie komunikatów o błędach walidacji?	Bezpośrednio przy błędnym polu (input), a nie zgrupowane w jednym miejscu (np. na górze strony). 
Dlaczego **nie należy** wyłączać (disable) przycisku &quot;Submit&quot; w formularzu, nawet jeśli jest on niekompletny?	Ponieważ użytkownik może nie wiedzieć, co jest nie tak lub przeoczyć pole
Dlaczego poleganie wyłącznie na kolorze (np. czerwona ramka) przy oznaczaniu błędów jest błędem UX?	Ponieważ osoby z zaburzeniami widzenia barw (daltoniści) mogą nie zauważyć błędu
Prawda czy Fałsz: Komunikaty o błędach powinny być ukryte pod ikoną (tooltip), aby nie zaśmiecać interfejsu.	Fałsz (błędy powinny być widoczne od razu, bez konieczności najeżdżania myszką).
Kiedy należy pokazać użytkownikowi wymagania dotyczące hasła (np. &quot;min. 8 znaków&quot;)?	Od razu (upfront), zanim zacznie pisać, a nie dopiero po wystąpieniu błędu. 
Jaki język powinien być stosowany w komunikatach błędów?	Ludzki i zrozumiały (human language), a nie techniczny (np. &quot;Database error 503&quot; vs &quot;Coś poszło nie tak, spróbuj ponownie&quot;).
Co jest lepsze od świetnego systemu raportowania błędów w formularzu?	Projektowanie formularza tak, aby zapobiegać błędom, zanim one wystąpią (Prevention over Cure).
Czy warto stosować &quot;positive feedback&quot; (zielone oznaczenia) w formularzach?	Tak, warto informować użytkownika nie tylko o błędach, ale też o poprawnym wypełnieniu pola (np. zielony &quot;ptaszek&quot;).
Do jakiego typu układów (wymiarów) został pierwotnie zaprojektowany Flexbox?	Do układów jednowymiarowych (1D) – rzędy LUB kolumny.
Do jakiego typu układów (wymiarów) został zaprojektowany CSS Grid?	Do układów dwuwymiarowych (2D) – rzędy I kolumny jednocześnie.
Jaka właściwość pozwala Flexboxowi na symulowanie układu dwuwymiarowego (przełamywanie linii)?	`flex-wrap`
Jeśli chcesz stworzyć układ kart, w którym wszystkie elementy mają idealnie **równe rozmiary** (szerokość i wysokość) bez skomplikowanych obliczeń, którego narzędzia najlepiej użyć?	CSS Grid. 
Prawda czy Fałsz: CSS Grid został wprowadzony, aby całkowicie zastąpić Flexboxa.	Fałsz (są to narzędzia komplementarne, często używane razem).
Jaka właściwość służąca do tworzenia odstępów między elementami była początkowo dostępna tylko w CSS Grid, a teraz jest wspierana również we Flexboxie?	gap
Czym inspirowany był model układu CSS Grid?	Układami siatki (grid layouts) stosowanymi w druku, np. w gazetach i czasopismach.
Czym różni się CSS Grid od Flexboxa w kontekście wymiarów układu?	CSS Grid jest dwuwymiarowy (wiersze i kolumny), a Flexbox jednowymiarowy (wiersz LUB kolumna).
Jak nazywamy element, na którym zastosowano `display: grid`?	Grid Container (Kontener siatki).
Jak nazywamy bezpośrednie dzieci kontenera siatki?	Grid Items (Elementy siatki).
Czym jest &quot;Grid Line&quot; (Linia siatki)?	To linia podziału tworząca strukturę siatki (może być pozioma lub pionowa). 
Czym jest &quot;Grid Track&quot; (Ścieżka siatki)?	To przestrzeń między dwiema sąsiednimi liniami siatki (czyli kolumna lub wiersz).
Czym jest &quot;Grid Area&quot; (Obszar siatki)?	To całkowita przestrzeń otoczona czterema liniami siatki (może składać się z wielu komórek). 
Czym jest &quot;Grid Cell&quot; (Komórka siatki)?	To najmniejsza jednostka siatki
Jaka jednostka w CSS Grid reprezentuje ułamek dostępnej wolnej przestrzeni?	{{c1::fr}}
Jaką funkcję CSS można użyć w `grid-template-columns` do zdefiniowania 12 równych kolumn bez wpisywania `1fr` 12 razy?	`repeat(12, 1fr)`
W właściwości `grid-template-areas`, jaki znak reprezentuje pustą komórkę siatki?	Kropka (`.`).
Jaki jest skrót (shorthand) dla właściwości `grid-template-rows`, `grid-template-columns` i `grid-template-areas`?	`grid-template` (lub po prostu `grid`).
Która właściwość wyrównuje elementy siatki wewnątrz ich komórek wzdłuż osi wiersza (inline/row axis)?	{{c1::justify-items}} 
Która właściwość wyrównuje elementy siatki wewnątrz ich komórek wzdłuż osi kolumny (block/column axis)?	{{c1::align-items}}
Jaki jest skrót dla `align-items` i `justify-items`?	`place-items`
Która właściwość wyrównuje **całą siatkę** wewnątrz kontenera wzdłuż osi wiersza (gdy siatka jest mniejsza niż kontener)?	{{c1::justify-content}}
Która właściwość wyrównuje **całą siatkę** wewnątrz kontenera wzdłuż osi kolumny?	{{c1::align-content}}
Co to jest &quot;Implicit Grid&quot; (Siatka niejawna)?	Siatka utworzona automatycznie przez przeglądarkę dla elementów, które nie mieszczą się w zdefiniowanej siatce jawnej (explicit grid).
Jakie właściwości określają rozmiar ścieżek utworzonych w siatce niejawnej?	`grid-auto-rows` oraz `grid-auto-columns`.
Co robi właściwość `grid-auto-flow: dense`?	Nakazuje algorytmowi automatycznego rozmieszczania wypełnianie pustych przestrzeni (dziur) w siatce mniejszymi elementami, które pojawiają się później w kodzie HTML. 
Jaka jest różnica między `grid-column-start` a `grid-column-end`?	`start` określa linię, na której element się zaczyna, a `end` linię, na której się kończy.
Jak sprawić, by element siatki rozciągał się na 3 kolumny, używając słowa kluczowego `span`?	`grid-column: span 3
Jaki jest skrót dla `grid-row-start` i `grid-row-end`?	`grid-row` (składnia: `start / end`).
Która właściwość pozwala elementowi dziecka nadpisać wyrównanie `justify-items` ustawione na rodzicu?	`justify-self`
Która właściwość pozwala elementowi dziecka nadpisać wyrównanie `align-items` ustawione na rodzicu?	`align-self`
Jaka funkcja definiuje zakres wielkości ścieżki (min i max), pozwalając jej na elastyczność?	`minmax(min, max)`
Jaka jest różnica między `auto-fill` a `auto-fit` w funkcji `repeat()`?	`auto-fill` zachowuje puste kolumny, jeśli jest na nie miejsce
Czym jest CSS Subgrid?	To funkcja pozwalająca zagnieżdżonej siatce (dziecku elementu grid) na dziedziczenie linii i ścieżek z siatki rodzica (używając `grid-template-columns: subgrid`).
Jak nazywamy przestrzeń w siatce ograniczoną dwiema sąsiednimi liniami (poziomymi lub pionowymi)?	Grid Track (Ścieżka siatki).
Jak nazywamy linie dzielące siatkę, które powstają automatycznie po zdefiniowaniu ścieżek?	Grid Lines (Linie siatki).
Od jakiej liczby zaczyna się numerowanie linii siatki?	Od 1.
Co to jest &quot;Grid Cell&quot; (Komórka siatki)?	To najmniejsza jednostka siatki,
 przecięcie jednej kolumny i jednego wiersza (jak w arkuszu kalkulacyjnym).	
Jakiej właściwości CSS używa się w elemencie dziecka, aby określić, na której linii pionowej ma się zaczynać?	`grid-column-start`
Jaki jest format skróconej właściwości `grid-column` (i `grid-row`)?	`start / end` (np. `2 / 4`).
Jeśli element ma zająć obszar od 1 do 3 linii wiersza i od 1 do 2 linii kolumny, jak zapisać to za pomocą skrótu `grid-area` (używając numerów linii)?	`grid-area: 1 / 1 / 3 / 2
Do czego służy właściwość `grid-area`, gdy przypisujemy jej wartość tekstową (np. `grid-area: header	`)?
Jaka właściwość **kontenera** pozwala na wizualne mapowanie układu strony za pomocą nazwanych obszarów?	`grid-template-areas` 
Jaki znak w `grid-template-areas` reprezentuje pustą komórkę siatki?	Kropka (`.`).
Jaka funkcja CSS pozwala zdefiniować wiele kolumn o tym samym rozmiarze bez konieczności wpisywania ich ręcznie?	`repeat()` (np. `repeat(5, 1fr)`).
Co reprezentuje jednostka `fr` w CSS Grid?	Ułamek **pozostałej** dostępnej przestrzeni w kontenerze (fraction of remaining space).
Jeśli zdefiniujesz kolumny jako `1fr 2fr`, jak zostanie podzielona przestrzeń?	Druga kolumna otrzyma dwa razy więcej miejsca niż pierwsza (odpowiednio 1/3 i 2/3 dostępnej przestrzeni).
Co zwróci funkcja CSS `min(100px, 50vw)`, jeśli okno przeglądarki ma szerokość 1000px (czyli 50vw = 500px)?	100px (ponieważ funkcja `min` zawsze wybiera **mniejszą** z podanych wartości).
Co zwróci funkcja CSS `max(100px, 50vw)`, jeśli okno przeglądarki ma szerokość 1000px?	500px (ponieważ funkcja `max` zawsze wybiera **większą** z podanych wartości).
Jaka funkcja CSS (specyficzna głównie dla Grida) pozwala zdefiniować zakres wielkości ścieżki od wartości minimalnej do maksymalnej?	`minmax(min, max)`
Jaka jest składnia funkcji `clamp()`, która pozwala na płynne skalowanie elementu w określonych granicach?	`clamp(minimum, ideal, maximum)`
Jaka jest kluczowa różnica między `auto-fit` a `auto-fill` w funkcji `repeat()`?	`auto-fit` **rozciąga** istniejące elementy, aby wypełniły cały wiersz (zwija puste ścieżki do 0), natomiast `auto-fill` **zachowuje** miejsce na potencjalne nowe kolumny (pozostawia puste przestrzenie), nawet jeśli ich nie ma.
Jak wygląda &quot;święty graal&quot; responsywnego Grida (snippet CSS), który tworzy tyle kolumn, ile się zmieści, bez użycia Media Queries?	`grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))
Czy funkcja `clamp()` może być używana tylko w CSS Grid?	Nie, `clamp()` jest uniwersalną funkcją CSS (działa np. dla `font-size`, `width` itd.), w przeciwieństwie do `minmax()`, która działa głównie w Gridzie.
Czy CSS Grid ma na celu zastąpienie Flexboxa?	Nie, są to narzędzia komplementarne: Grid do układów dwuwymiarowych, Flexbox do jednowymiarowych.
Które narzędzie jest lepsze do projektowania &quot;Content-First&quot; (gdzie układ wynika z wielkości i zachowania treści)?	Flexbox (pozwala elementom elastycznie rosnąć/maleć w zależności od ich zawartości).
Które narzędzie jest lepsze do projektowania &quot;Layout-First&quot; (gdzie najpierw definiujesz sztywną strukturę wierszy/kolumn, a potem wlewasz w nią treść)?	CSS Grid (pozwala na precyzyjne rozmieszczenie elementów w zdefiniowanych ścieżkach).
Jeśli potrzebujesz wyrównać elementy tylko w jednej linii (wierszu LUB kolumnie), które narzędzie jest zazwyczaj bardziej intuicyjne?	Flexbox.
Jeśli potrzebujesz precyzyjnie rozmieścić elementy w dwóch wymiarach jednocześnie (wiersze I kolumny), które narzędzie wybrać?	CSS Grid.
Jaka jest popularna strategia łączenia Grida i Flexboxa w jednym projekcie?	Użycie Grida do ogólnego układu strony (makro-layout: nagłówek, sidebar, główna treść) oraz Flexboxa do układania elementów wewnątrz poszczególnych komórek (mikro-layout: np. nawigacja, karty). 
Co oznacza, że Flexbox jest jednowymiarowy (1D)?	Że zarządza układem tylko w jednej osi na raz (oś główna - wiersz lub kolumna), a zawijanie (wrap) tworzy po prostu nowe, niezależne linie flex.
