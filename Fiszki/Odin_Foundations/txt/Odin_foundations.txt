Co to jest Internet?	Globalna sieć miliardów komputerów i innych urządzeń elektronicznych, które mogą się ze sobą komunikować.
Co to jest sieć (Network)?	Grupa dwóch lub więcej komputerów połączonych ze sobą w celu współdzielenia zasobów.
Co to są pakiety (Packets)?	Małe fragmenty, na które dzielone są dane przesyłane przez internet. Są wysyłane niezależnie i ponownie składane w całość u odbiorcy.
Co to jest Serwer (Server)?	Komputer podłączony do internetu, który przechowuje pliki stron internetowych i danych, a następnie wysyła je do klienta na żądanie.
Co to jest Klient (Client)?	Urządzenie (np. laptop, telefon) lub oprogramowanie (przeglądarka), które wysyła żądanie do serwera w celu uzyskania dostępu do danych.
Co to jest Serwer WWW (Web Server)?	Specjalny rodzaj serwera (oprogramowanie i sprzęt), który przechowuje pliki strony internetowej i udostępnia je przeglądarkom.
Co to jest adres IP?	Unikalny ciąg liczb (np. 172.16.254.1) przypisany do każdego urządzenia w sieci, pozwalający na jego identyfikację i lokalizację.
Co to jest DNS (Domain Name System)?	System działający jak &quot;książka telefoniczna&quot; internetu
Co to jest Przeglądarka (Web Browser)?	Oprogramowanie (np. Chrome, Firefox) służące do lokalizowania, pobierania i wyświetlania stron internetowych.
Co to jest Wyszukiwarka (Search Engine)?	Strona internetowa (np. Google, Bing), która pomaga użytkownikom znajdować inne strony internetowe w sieci.
Co to jest Router?	Urządzenie sprzętowe, które kieruje ruchem danych (pakietami) między różnymi sieciami (np. między Twoją siecią domową a internetem).
Co to jest ISP?	Internet Service Provider (Dostawca usług internetowych) – firma, która zapewnia dostęp do internetu.
Jaka jest różnica między stroną WWW a serwerem WWW?	Strona to dokument/plik, który czytasz. Serwer to komputer, na którym ten plik jest przechowywany.
Co dzieje się po wpisaniu adresu w przeglądarce (uproszczone)?	Przeglądarka pyta serwer DNS o adres IP domeny -&gt; DNS zwraca IP -&gt; Przeglądarka wysyła żądanie do serwera pod tym IP -&gt; Serwer odsyła pakiety danych strony -&gt; Przeglądarka składa je w całość i wyświetla.
Co to jest Command Line (CLI)?	Interfejs tekstowy, w którym wpisujesz polecenia, które komputer ma wykonać.
Co oznacza symbol znaku zachęty (Prompt) - $ lub %?	Oznacza, że terminal oczekuje na wpisanie polecenia.
Dlaczego hasła nie są widoczne podczas wpisywania ich w terminalu (np. przy sudo)?	Jest to funkcja bezpieczeństwa, aby ukryć długość i treść hasła przed osobami postronnymi.
Jaki jest skrót klawiszowy do KOPIOWANIA w terminalu (Linux/Windows)?	Ctrl + Shift + C (na macOS: Cmd + C)
Jaki jest skrót klawiszowy do WKLEJANIA w terminalu (Linux/Windows)?	Ctrl + Shift + V (na macOS: Cmd + V)
Co robi klawisz TAB w terminalu?	Automatycznie uzupełnia polecenia lub ścieżki do plików (Tab completion).
Do czego służy polecenie `whoami`?	Zwraca nazwę zalogowanego użytkownika.
Do czego służy polecenie `pwd`?	Wyświetla ścieżkę do katalogu, w którym aktualnie się znajdujesz (Print Working Directory).
Do czego służy polecenie `ls`?	Wyświetla zawartość (pliki i foldery) w bieżącym katalogu.
Do czego służy polecenie `cd` (Change Directory)?	Służy do nawigacji (przechodzenia) między folderami.
Gdzie przeniesie Cię samo polecenie `cd` (bez argumentów) lub `cd ~`?	Do katalogu domowego użytkownika (Home directory).
Gdzie przeniesie Cię polecenie `cd ..`?	Poziom wyżej, do katalogu nadrzędnego (Parent directory).
Jakie polecenie tworzy nowy katalog (folder)?	`mkdir &lt;nazwa_folderu&gt;`
Jakie polecenie tworzy nowy, pusty plik?	`touch &lt;nazwa_pliku&gt;`
Jak usunąć plik w terminalu?	`rm &lt;nazwa_pliku&gt;`
Jak zmienić nazwę pliku lub katalogu (lub go przenieść)?	`mv &lt;stara_nazwa&gt; &lt;nowa_nazwa&gt;`
Co oznacza kropka `.` w poleceniu takim jak `code .`?	Oznacza &quot;bieżący katalog&quot;.
Jak otworzyć bieżący folder w VS Code z poziomu terminala?	`code .`
Co to jest Git?	System kontroli wersji (Version Control System). Pozwala śledzić zmiany w plikach i zarządzać historią projektu.
Jaka jest różnica w &quot;zapisywaniu&quot; między edytorem tekstu a Gitem?	Edytor nadpisuje plik (masz tylko aktualną wersję). Git zapisuje różnice (zmiany) oraz zachowuje pełną historię wszystkich poprzednich wersji.
Czy Git działa lokalnie czy zdalnie?	Lokalnie (na Twoim komputerze). Nie wymaga internetu do działania.
Czy GitHub działa lokalnie czy zdalnie?	Zdalnie. To serwis w chmurze (w internecie) do przechowywania repozytoriów.
Co to jest GitHub?	Platforma internetowa służąca do hostowania projektów Git. Umożliwia współpracę, udostępnianie kodu i budowanie portfolio.
Dlaczego Git jest przydatny dla pojedynczego programisty?	Pozwala przeglądać historię rozwoju projektu i łatwo przywracać wcześniejsze wersje plików w razie błędu.
Dlaczego Git/GitHub są ważne dla zespołów?	Umożliwiają łączenie (merge) pracy wielu osób nad tym samym projektem i rozwiązywanie konfliktów w kodzie.
Co oznacza, że Git jest &quot;rozproszonym&quot; (distributed) systemem kontroli wersji?	Oznacza to, że każdy deweloper posiada na swoim dysku pełną kopię całego projektu wraz z jego historią, a nie tylko pliki nad którymi pracuje.
Jak sprawdzić status repozytorium (które pliki są zmienione/dodane)?	git status
Jak zainicjować (stworzyć) nowe repozytorium Git w obecnym folderze?	git init
"Jak dodać WSZYSTKIE zmienione pliki do ""poczekalni"" (staging area)?"	git add .
"Jak dodać TYLKO JEDEN konkretny plik do ""poczekalni""?"	git add <nazwa_pliku>
Jak stworzyć commit (trwały zapis) z wiadomością?	"git commit -m ""Twoja wiadomość"""
Jak wyświetlić historię commitów?	git log
Jak połączyć lokalne repozytorium z nowym repozytorium na GitHubie?	git remote add origin <link_do_repozytorium_na_githubie>
"Jak zmienić nazwę obecnej gałęzi na ""main"" (standard GitHub)?"	git branch -M main
Jak wysłać (wypchnąć) zmiany na GitHub (pierwszy raz dla gałęzi main)?	git push -u origin main
Jak wysłać zmiany na GitHub (kolejne razy)?	git push
Jak pobrać repozytorium z GitHuba na swój komputer?	git clone <link_do_repozytorium>
Co robi komenda `git add`?	Przenosi zmiany z katalogu roboczego (Working Directory) do poczekalni (Staging Area), przygotowując je do commita.
Co robi komenda `git commit`?	Bierze pliki z poczekalni (Staging Area) i zapisuje je w historii repozytorium jako nową migawkę (snapshot).
Co to jest &quot;Atomic Commit&quot; (Atomowy commit)?	To commit, który zawiera zmiany dotyczące tylko jednego zadania lub funkcjonalności. Ułatwia to cofanie zmian i pisanie czytelnych historii.
Dlaczego warto robić &quot;Atomic Commits&quot;?	1. Łatwiej cofnąć konkretną zmianę bez psucia innych. 2. Historia zmian jest czytelna dla innych programistów.
W poleceniu `git push origin main`, co oznacza &quot;origin&quot;?	To domyślna nazwa zdalnego repozytorium (połączenia z GitHubem).
W poleceniu `git push origin main`, co oznacza &quot;main&quot;?	To nazwa gałęzi (branch), którą wysyłasz na serwer.
Co oznacza plik wyświetlony na CZERWONO po wpisaniu `git status`?	Plik został zmieniony, ale nie jest jeszcze w poczekalni (staging area). Git o nim wie, ale go nie &quot;pilnuje&quot; w tym commicie.
Co oznacza plik wyświetlony na ZIELONO po wpisaniu `git status`?	Plik znajduje się w poczekalni (staging area) i zostanie dołączony do najbliższego commita.
Jaką komendą ustawić VS Code jako domyślny edytor dla komunikatów Gita (żeby nie otwierał się Vim)?	git config --global core.editor &quot;code --wait&quot;
Dlaczego NIE należy edytować plików bezpośrednio na stronie GitHub?	Ponieważ powoduje to konflikty wersji między Twoim komputerem a serwerem, które mogą być trudne do naprawienia dla początkujących.
Co robi komenda `git remote -v`?	Wyświetla listę zdalnych repozytoriów (URL) podłączonych do Twojego projektu (np. adres na GitHubie).
Co to jest HTML Tag (Znacznik)?	Słowo kluczowe zamknięte w nawiasach ostrych (np. <p>). Definiuje strukturę, ale sam w sobie nie jest pełnym elementem.
Jaka jest różnica między Znacznikiem (Tag) a Elementem (Element)?	Element to całość: znacznik otwierający + treść w środku + znacznik zamykający (np. <p>Tekst</p>). Znacznik to tylko <p>.
"Co to jest ""Void Element"" (Element pusty) i podaj przykład?"	Element, który nie ma treści ani znacznika zamykającego. Np. <br> (złamanie linii) lub <img> (obrazek).
"Co oznacza termin ""Semantic HTML"" (Semantyczny HTML)?"	Używanie znaczników zgodnie z ich przeznaczeniem i znaczeniem (np. <h1> dla nagłówka), a nie tylko dla wyglądu. Pomaga to wyszukiwarkom i czytnikom ekranowym.
Dlaczego główny plik strony powinien nazywać się `index.html`?	Ponieważ serwery domyślnie szukają pliku o tej nazwie, gdy użytkownik wchodzi na adres strony bez podania konkretnego pliku.
Do czego służy deklaracja `<!DOCTYPE html>` na początku pliku?	Informuje przeglądarkę, że używamy najnowszej wersji HTML (HTML5) i jak ma renderować dokument.
Co znajduje się w elemencie `<html>`?	Wszystkie inne elementy strony. Jest to tzw. element korzeń (root element).
Co znajduje się w elemencie `<head>`?	Metadane, tytuł strony, linki do stylów – rzeczy ważne dla przeglądarki, ale NIE wyświetlane bezpośrednio w oknie treści.
"Dlaczego warto zawsze dodawać `<meta charset=""UTF-8"">`?"	Zapewnia poprawne kodowanie znaków, dzięki czemu polskie znaki (ą, ę) i symbole specjalne wyświetlają się poprawnie.
Do czego służy element `<title>`?	Ustawia tytuł strony widoczny na karcie przeglądarki (w tabie) oraz w wynikach wyszukiwania Google.
Co znajduje się w elemencie `<body>`?	Wszystko to, co widzi użytkownik na stronie (tekst, obrazki, linki).
Jaki jest skrót w VS Code do wygenerowania całego szablonu HTML?	Wpisz wykrzyknik `!` i naciśnij Enter.
Co robi znacznik `<h1>` do `<h6>`?	Definiuje nagłówki. <h1> to najważniejszy nagłówek (tytuł strony), a <h6> to najmniej ważny.
Jakiego znacznika używamy do tworzenia akapitów tekstu?	<p>
Co robi znacznik `<strong>` i czym różni się od pogrubienia?	"Pogrubia tekst I dodaje mu znaczenie semantyczne ""ważne"" (important). Czytniki ekranowe mogą przeczytać go innym tonem."
Co robi znacznik `<em>` i czym różni się od kursywy?	"Pochyla tekst (Italic) I dodaje mu ""nacisk/emfazę"" (emphasis)."
Co to jest Zagnieżdżanie (Nesting)?	Umieszczanie jednego elementu wewnątrz drugiego (tworzenie relacji Rodzic-Dziecko).
Jak wygląda składnia komentarza w HTML?	
Jaki jest skrót w VS Code do zrobienia komentarza?	Ctrl + / (lub Cmd + / na Macu)
Jak przeglądarka traktuje wielokrotne spacje i nowe linie w kodzie HTML (wewnątrz tekstu)?	Ignoruje je i zamienia na jedną spację (tzw. whitespace collapsing).
Jaki znacznik służy do tworzenia listy nieuporządkowanej (z kropkami)?	<ul> (Unordered List)
Jaki znacznik służy do tworzenia listy uporządkowanej (z numerami)?	<ol> (Ordered List)
Jaki znacznik oznacza pojedynczy element listy (zarówno w <ul> jak i <ol>)?	<li> (List Item)
Jaki znacznik służy do tworzenia linków (kotwic)?	<a> (Anchor)
Jaki atrybut w znaczniku `<a>` określa adres docelowy linku?	href
"Napisz kod: Link prowadzący do strony google.com z tekstem ""Google"""	"<a href=""https://google.com"">Google</a>"
Jaki atrybut sprawia, że link otwiera się w nowej karcie?	"target=""_blank"""
"Dlaczego przy używaniu `target=""_blank""` należy dodać `rel=""noopener noreferrer""`?"	Ze względów bezpieczeństwa (zapobiega atakom typu phishing/tabnabbing) i prywatności.
"Czym jest ""Link Absolutny"" (Absolute link)?"	To pełny adres internetowy zawierający protokół i domenę (np. https://google.com). Prowadzi zazwyczaj do zewnętrznych stron.
"Czym jest ""Link Relatywny"" (Relative link)?"	To ścieżka do pliku względem pliku, w którym się znajdujesz (np. ./strona.html). Nie zawiera domeny. Służy do linkowania wewnątrz własnej strony.
Co oznacza kropka i ukośnik `./` na początku ścieżki (np. `./obrazek.jpg`)?	"Oznacza ""obecny katalog"" (ten sam folder, w którym jest plik HTML)."
Co oznaczają dwie kropki i ukośnik `../` na początku ścieżki?	"Oznacza ""katalog nadrzędny"" (wyjdź jeden folder wyżej/wstecz)."
Jaki znacznik służy do wstawiania obrazków?	<img>
Czy znacznik `<img>` wymaga znacznika zamykającego?	"Nie, jest to ""void element"" (element pusty)."
Jaki atrybut wskazuje źródło pliku obrazka?	src
Do czego służy atrybut `alt` w obrazkach?	Zawiera tekst alternatywny wyświetlany, gdy obrazek się nie załaduje. Jest też czytany przez czytniki dla osób niewidomych.
Dlaczego warto podawać atrybuty `width` i `height` w obrazkach, nawet jeśli zmieniamy je w CSS?	"Aby zapobiec ""skakaniu"" układu strony (layout shift) podczas ładowania obrazka."
Wymień 4 główne formaty obrazków używane w internecie (wg Knowledge Check)	JPEG, PNG, GIF, SVG
"Napisz kod: Obrazek ""kot.jpg"" znajdujący się w folderze ""img"" (względem obecnego pliku) z opisem ""Mój kot"""	"<img src=""./img/kot.jpg"" alt=""Mój kot"">"
Dlaczego warto dbać o jakość wiadomości commitów (wymień 2 powody)?	1. Pomaga innym (i Tobie w przyszłości) zrozumieć historię zmian i naprawiać błędy. 2. Rekruterzy patrzą na jakość commitów przy zatrudnianiu jako dowód profesjonalizmu.
Z jakich dwóch głównych części składa się profesjonalna wiadomość commita?	Z Tematu (Subject) i Treści (Body).
Co powinno oddzielać Temat (Subject) od Treści (Body) w wiadomości commita?	Pusta linia (blank line).
Jaki jest zalecany limit znaków dla Tematu (Subject) commita i dlaczego?	Około 72 znaki. GitHub ucina dłuższe tematy w widoku listy.
Co powinno znaleźć się w Treści (Body) commita?	Wyjaśnienie DLACZEGO zmiana była potrzebna i JAK problem został rozwiązany (kontekst), a nie tylko &quot;co&quot; zrobiono.
Dlaczego wiadomość commita typu &quot;fix bug&quot; albo &quot;update&quot; jest uważana za złą?	Jest zbyt ogólna. Nie mówi, co dokładnie naprawiono ani dlaczego, co wprowadza zamieszanie w zespole.
W jakim trybie gramatycznym (po angielsku) powinno się pisać Temat commita?	W trybie rozkazującym (Imperative Mood), np. &quot;Add feature&quot; zamiast &quot;Added feature&quot; czy &quot;Adding feature&quot;.
Jak wywołać edytor tekstowy (np. VS Code) do napisania długiego commita (z Subject i Body), zamiast pisać go w cudzysłowie?	Wpisz w terminalu samo `git commit` (bez flagi `-m`).
Kiedy należy zrobić commit?	Po każdej &quot;znaczącej zmianie&quot; (np. gdy fragment kodu zadziała, naprawisz błąd lub literówkę). Zasada: &quot;Commit early and often&quot; (Wcześnie i często).
Czym jest commit w kontekście historii projektu (metafora użyta w lekcji)?	Jest &quot;migawką&quot; (snapshot) stanu kodu w danym momencie, do której zawsze można wrócić.
Z czego składa się reguła CSS?	Z selektora i bloku deklaracji (w nawiasach klamrowych).
Napisz kod: Selektor uniwersalny (wybierający wszystkie elementy)	*
Jaki znak reprezentuje selektor KLASY?	. (kropka)
Jaki znak reprezentuje selektor ID?	"# (hash / płotek)"
Czym różni się Klasa od ID?	Klasę można użyć na wielu elementach wielokrotnie. ID musi być unikalne (tylko jeden taki element na stronie).
Czy nazwa klasy lub ID może zaczynać się od cyfry?	Nie, to niepoprawna nazwa w CSS.
Napisz kod: Selektor grupowy dla elementów h1 oraz p (ten sam styl dla obu)	h1, p { ... }
Co robi selektor `div.alert` (brak spacji)?	Wybiera tylko te elementy &lt;div&gt;, które JEDNOCZEŚNIE mają klasę &quot;alert&quot; (Chaining).
Co robi selektor `.menu .item` (jest spacja)?	Wybiera elementy z klasą &quot;item&quot;, które są POTOMKAMI (są wewnątrz) elementu z klasą &quot;menu&quot;. (Descendant combinator).
Napisz kod: Selektor potomka (wybierz &lt;p&gt; będące wewnątrz &lt;div&gt;)	div p { ... }
Napisz kod: Linkowanie zewnętrznego pliku CSS w HTML	&lt;link rel=&quot;stylesheet&quot; href=&quot;style.css&quot;&gt;
Gdzie umieszczamy element `&lt;link&gt;` do pliku CSS?	W sekcji &lt;head&gt;
Jak dodać CSS &quot;wewnętrznie&quot; (Internal CSS) bez osobnego pliku?	Używając znacznika &lt;style&gt; w sekcji &lt;head&gt;.
Jak dodać CSS &quot;liniowo&quot; (Inline CSS) bezpośrednio do elementu?	Używając atrybutu style=&quot;...&quot; wewnątrz znacznika otwierającego.
Która metoda dodawania CSS jest najlepsza i dlaczego?	Zewnętrzny plik CSS (External). Oddziela treść od wyglądu, odchudza kod HTML i pozwala używać jednego pliku stylów dla wielu podstron.
Co się stanie, jeśli obrazek nie ma zdefiniowanej wysokości i szerokości w HTML/CSS?	Może spowodować &quot;skakanie&quot; treści strony podczas ładowania, ponieważ przeglądarka nie zarezerwuje na niego miejsca przed pobraniem.
Jak zachować proporcje obrazka, zmieniając tylko jego szerokość?	Ustawić `width` na konkretną wartość, a `height` na `auto`.
Co oznacza `font-family: &quot;Arial&quot;, sans-serif	` (dlaczego są dwie wartości)?
Jaka wartość liczbowa odpowiada `font-weight: bold`?	700
Co to jest &quot;Deklaracja&quot; (Declaration) w regule CSS?	To pojedyncza para: właściwość i wartość (np. `color: red`), zakończona średnikiem.
Czy w HTML można nadać elementowi więcej niż jedną klasę?	Tak, oddzielając je spacją (np. class=&quot;btn btn-danger&quot;).
Jak w CSS wybrać element, który posiada D WIE konkretne klasy jednocześnie (np. &quot;btn&quot; i &quot;danger&quot;)?	.btn.danger (bez spacji między kropkami)
Dlaczego nie można łączyć w łańcuch dwóch selektorów typu (np. `divp`)?	Ponieważ jeden element HTML nie może być jednocześnie dwoma różnymi tagami (nie może być naraz divem i paragrafem).
Czy nazwy klas i ID w CSS są wrażliwe na wielkość liter (Case Sensitive)?	Tak, `.Alert` i `.alert` to dwie różne klasy.
Wymień 3 sposoby definiowania kolorów w CSS (poza nazwami słownymi).	HEX (szesnastkowy), RGB, HSL.
"Co oznacza format koloru `#FF0000`?"	To format HEX (szesnastkowy).
Co robi właściwość `font-family`?	Określa krój czcionki (fontu).
Kiedy nazwa czcionki w `font-family` MUSI być ujęta w cudzysłów?	Gdy nazwa zawiera spację (np. &quot;Times New Roman&quot;).
Czy nazwy ogólne czcionek (generic family names), np. sans-serif, serif, piszemy w cudzysłowie?	Nie, nazwy ogólne piszemy bez cudzysłowu.
Jaka jest domyślna wartość `font-weight: normal` wyrażona liczbowo?	400
Jaka właściwość odpowiada za wyrównanie tekstu w poziomie (np. do środka)?	text-align
Co się stanie, jeśli w selektorze potomka zrobisz `.rodzic .dziecko`, ale `.dziecko` jest zagnieżdżone bardzo głęboko (np. jest prawnukiem)?	Zadziała. Selektor potomka (spacja) działa niezależnie od głębokości zagnieżdżenia.
Jak w CSS ustawić tło elementu?	background-color

Który sposób definiowania funkcji jest zalecany jako pierwszy wybór i dlaczego?	Function Declaration. Jest bardziej czytelna i pozwala na wywołanie funkcji przed jej definicją (daje swobodę w organizacji kodu).
Jaki jest najczęstszy błąd początkujących programistów wg V. Antona Spraula?	Skupianie się na nauce składni (syntax) zamiast na nauce rozwiązywania problemów.

Jak powinny zaczynać się nazwy zmiennych przechowujących dane (np. stringi, liczby)?	Od rzeczownika lub przymiotnika (np. `userName`, `isActive`, `playerScore`).

Jaka jest główna zasada dotycząca komentarzy w kodzie (&quot;Why vs How&quot;)?	Komentarze powinny wyjaśniać DLACZEGO coś zostało zrobione (intencję, biznesowy powód), a nie JAK działa kod (to powinno wynikać z samej nazwy funkcji/zmiennej).

Czy metody `map`, `filter` i `reduce` zmieniają oryginalną tablicę (mutate)?	Nie. Wszystkie zwracają nową wartość/tablicę, a oryginał pozostaje bez zmian (Non-mutating).

Dlaczego używamy TDD (podaj jedną zaletę)?	Daje pewność, że kod działa poprawnie i ułatwia późniejsze zmiany (refactoring), bo testy od razu wykryją błędy.

Co to jest &quot;Callback&quot; w kontekście `addEventListener`?	Funkcja, którą przekazujemy jako drugi argument, a która zostanie wywołana (&quot;called back&quot;) dopiero wtedy, gdy zdarzenie nastąpi.
Co to jest parametr `e` (lub `event`) w funkcji callback?	To obiekt zdarzenia, który zawiera szczegóły na temat tego, co się stało (np. `e.target` mówi, co dokładnie zostało kliknięte, `e.key` jaki klawisz wciśnięto).
Co to jest &quot;Event Bubbling&quot; (Bąbelkowanie Zdarzeń)?	Mechanizm, w którym zdarzenie (np. kliknięcie) najpierw odpala się na elemencie docelowym, a potem &quot;bąbelkuje&quot; w górę do rodzica, dziadka, aż do `document`.
Jak zatrzymać bąbelkowanie zdarzenia (żeby nie dotarło do rodziców)?	`e.stopPropagation()`
Gdzie w pliku HTML najlepiej umieścić tag `&lt;script&gt;` i dlaczego?	Na końcu `&lt;body&gt;` (przed zamknięciem), aby mieć pewność, że cały DOM został wczytany przed uruchomieniem skryptu (lub w `&lt;head&gt;` z atrybutem `defer`).
Jaka jest różnica między `element.children` a `element.childNodes`?	`children` zwraca tylko ELEMENTY HTML (tagi). `childNodes` zwraca WSZYSTKIE węzły, w tym węzły tekstowe (np. białe znaki/entery między tagami) i komentarze.
Jaka jest różnica między `nextSibling` a `nextElementSibling`?	`nextSibling` może zwrócić węzeł tekstowy (np. enter). `nextElementSibling` zwróci następny konkretny element HTML (tag).
Jak dostać się do rodzica danego elementu?	Używając właściwości `parentElement` (lub `parentNode`).
Mamy w HTML: `&lt;div data-key=&quot;123&quot; data-user-name=&quot;Bob&quot;&gt;`. Jak dostać się do tych danych w JS?	Używając właściwości `dataset`: `element.dataset.key` oraz `element.dataset.userName` (zauważ zmianę na camelCase!).
Czy `element.style.color` zwróci kolor zdefiniowany w arkuszu CSS (pliku .css)?	Nie. `element.style` zwraca tylko style wpisane &quot;inline&quot; (bezpośrednio w tagu HTML).
Jak odczytać OSTATECZNY styl elementu (nawet ten z pliku CSS), jaki widzi przeglądarka?	Używając metody `window.getComputedStyle(element).color`
Jaka jest różnica między `textContent` a `innerText`?	`textContent` zwraca CAŁY tekst, nawet ten ukryty przez CSS. `innerText` zwraca tylko tekst widoczny dla użytkownika (szanuje style CSS) i jest wolniejszy.
Co to jest `DocumentFragment` i do czego służy?	To &quot;lekki&quot;, wirtualny kontener na elementy DOM. Używamy go, gdy chcemy dodać wiele elementów do strony na raz (np. w pętli), aby uniknąć wielokrotnego przerysowywania strony (Reflow).
Jaka jest różnica między &quot;Live Collection&quot; (np. z `getElementsByClassName`) a &quot;Static Collection&quot; (np. z `querySelectorAll`)?	Żywa kolekcja aktualizuje się automatycznie, gdy DOM się zmienia (dodasz nowy element - kolekcja rośnie). Statyczna to &quot;zdjęcie&quot; stanu w momencie wywołania (nie zmienia się).
Jak sprawdzić, czy element posiada konkretną klasę CSS (bez manualnego przeszukiwania stringa)?	"`element.classList.contains(&#x27;nazwa-klasy&#x27;)`"
Jak przełączyć klasę (dodać jak nie ma, usunąć jak jest) jedną metodą?	"`element.classList.toggle(&#x27;nazwa-klasy&#x27;)`"
Dlaczego operacje zapisu do DOM (np. zmiana wymiarów, dodawanie elementów) są &quot;kosztowne&quot;?	Ponieważ zmuszają przeglądarkę do przeliczenia układu strony (Reflow/Layout) i ponownego narysowania pikseli (Repaint).
Co reprezentuje obiekt `window` w JavaScript?	Reprezentuje całe okno przeglądarki (lub kartę), w którym załadowana jest strona. Zawiera właściwości takie jak szerokość okna (`innerWidth`) i jest &quot;rodzicem&quot; dla obiektu `document`.
Co reprezentuje obiekt `navigator`?	Reprezentuje tożsamość i stan samej przeglądarki (nie strony). Używany np. do sprawdzania geolokalizacji, języka użytkownika czy dostępu do kamery.
Co reprezentuje obiekt `document`?	Reprezentuje konkretną stronę załadowaną w oknie (DOM). To główny punkt wejścia do manipulacji treścią HTML i stylami.
Jaka jest różnica między &quot;Child Node&quot; (Dziecko) a &quot;Descendant Node&quot; (Potomek)?	Dziecko jest BEZPOŚREDNIO wewnątrz rodzica (1 poziom niżej). Potomek to dowolny węzeł wewnątrz (dziecko, wnuk, prawnuk itd.).
Co zwraca `document.querySelector(&quot;p&quot;)`?	Zwraca referencję do PIERWSZEGO elementu `&lt;p&gt;` znalezionego w dokumencie.
Co zwraca `document.querySelectorAll(&quot;p&quot;)`?	Zwraca `NodeList` (obiekt tablicopodobny) zawierający referencje do WSZYSTKICH elementów `&lt;p&gt;` w dokumencie.
Jak zmienić tekst wewnątrz elementu (np. w linku) używając JS?	Używając właściwości `textContent`, np. `link.textContent = &quot;Nowy Tekst&quot;
Jak stworzyć nowy element HTML (np. paragraf) w pamięci?	`const para = document.createElement(&quot;p&quot;)
Czy `document.createElement()` automatycznie dodaje element do widocznej strony?	Nie. Tworzy go tylko w pamięci. Aby był widoczny, trzeba go dodać do drzewa DOM (np. używając `appendChild`).
Co się stanie, jeśli użyjesz `appendChild()` na elemencie, który JUŻ istnieje w innym miejscu na stronie?	Element zostanie PRZENIESIONY (Moved) w nowe miejsce, a nie skopiowany. W DOM może istnieć tylko jedna instancja danego węzła.
Jak usunąć element, mając do niego referencję (w nowoczesnych przeglądarkach)?	`element.remove()
Dlaczego w JS używamy `style.backgroundColor` zamiast `style.background-color`?	Ponieważ w JavaScript myślnik `-` jest operatorem odejmowania. Właściwości CSS z myślnikami są konwertowane na `camelCase`.
Jaka jest zalecana metoda zmiany wyglądu elementu zamiast bezpośredniej edycji `element.style`?	Dodawanie lub usuwanie klas CSS za pomocą `element.classList.add(&quot;nazwa-klasy&quot;)`. Jest to czystsze i oddziela style od logiki.
Co robi metoda `event.preventDefault()` (użyta np. przy wysyłaniu formularza)?	Zapobiega domyślnej akcji przeglądarki. W przypadku formularza zapobiega przeładowaniu strony po kliknięciu przycisku submit.
Co robi metoda `input.focus()`?	Ustawia kursor w danym polu tekstowym, gotowy do pisania.
