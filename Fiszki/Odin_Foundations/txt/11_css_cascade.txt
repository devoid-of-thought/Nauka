Co to jest Kaskada (The Cascade) w CSS?;To algorytm określający, która reguła zostanie zastosowana do elementu, gdy istnieje wiele sprzecznych reguł (np. jedna mówi "czerwony", a druga "niebieski").;odin_css_cascade
Wymień 3 główne czynniki, które Kaskada bierze pod uwagę (w kolejności ważności).;1. Specyficzność (Specificity), 2. Dziedziczenie (Inheritance), 3. Kolejność reguł (Rule Order).;odin_css_cascade
Uporządkuj selektory od NAJMNIEJ do NAJBARDZIEJ specyficznego: Klasa, ID, Element (Type).;Element (Type) < Klasa < ID;odin_css_cascade
Co ma wyższą wagę (specyficzność): jeden selektor ID czy 10 selektorów Klasy?;Jeden selektor ID (ID zawsze wygrywa z dowolną liczbą klas).;odin_css_cascade
Co ma wyższą wagę (specyficzność): jeden selektor Klasy czy 10 selektorów Elementu (Tagów)?;Jeden selektor Klasy (Klasa zawsze wygrywa z dowolną liczbą tagów).;odin_css_cascade
Mamy reguły: `#header { color: red; }` oraz `.menu .item { color: blue; }`. Jaki będzie kolor?;Czerwony. Reguła pierwsza ma ID, które jest silniejsze niż dwie klasy.;odin_css_cascade
Mamy reguły: `.nav .item { color: red; }` oraz `.item { color: blue; }`. Jaki będzie kolor?;Czerwony. Pierwsza reguła ma dwie klasy, druga tylko jedną (2 > 1).;odin_css_cascade

Czy selektor uniwersalny `*` dodaje jakąkolwiek wagę do specyficzności?;Nie, nie ma żadnej wartości specyficzności (jest słabszy nawet od selektora typu).;odin_css_cascade
Czy kombinatory (takie jak `+`, `>`, `~` lub spacja) zwiększają specyficzność reguły?;Nie. Liczą się tylko same selektory (klasy, ID, tagi), a nie znaki między nimi.;odin_css_cascade
Co to jest Dziedziczenie (Inheritance)?;Przekazywanie niektórych właściwości CSS z elementu rodzica na elementy dzieci (np. kolor tekstu).;odin_css_cascade
