Odin_javascript::10_oop_principles	Co to jest Zasada Pojedynczej Odpowiedzialności (Single Responsibility Principle - SRP)?	Mówi ona, że klasa (obiekt, moduł) powinna mieć tylko jedną odpowiedzialność, co oznacza, że powinna mieć tylko **jeden powód do zmiany**. 	
Odin_javascript::10_oop_principles	Dlaczego mieszanie logiki aplikacji (np. sprawdzania warunku końca gry) z manipulacją DOM w jednej funkcji łamie zasadę SRP?	Ponieważ funkcja ma wtedy dwie odpowiedzialności: obliczanie stanu gry ORAZ aktualizację interfejsu użytkownika. Należy to rozdzielić na dwa niezależne moduły.	
Odin_javascript::10_oop_principles	Czym są "ściśle powiązane" (tightly coupled) obiekty?	To obiekty, które tak mocno na sobie polegają (wiedzą o sobie za dużo), że zmiana lub usunięcie jednego z nich wymusza całkowitą przebudowę drugiego. 	
Odin_javascript::10_oop_principles	Jaka jest główna zaleta tworzenia "luźno powiązanych" (loosely coupled) obiektów w aplikacji?	Pozwalają na łatwe i bezpieczne modyfikacje. Na przykład można całkowicie zmienić sposób działania interfejsu użytkownika (UI/DOM) bez konieczności dotykania i psucia głównej logiki biznesowej (np. zasad gry).	
Odin_javascript::10_oop_principles	Czym jest akronim SOLID?	To zestaw pięciu podstawowych zasad projektowania zorientowanego obiektowo (OOP), pomagających w tworzeniu łatwiejszego w utrzymaniu i skalowaniu kodu. Pierwszą z nich jest Single Responsibility (S). 	
Odin_javascript::10_oop_principles	Zgodnie z popularnymi i nowoczesnymi wzorcami projektowymi, co jest zazwyczaj preferowane: Kompozycja (Composition) czy Dziedziczenie (Inheritance)?	Zazwyczaj preferowana jest **Kompozycja** (favoring composition over inheritance), ponieważ budowanie obiektów z mniejszych, niezależnych części zapewnia większą elastyczność i pozwala uniknąć problemów ze sztywnymi, głębokimi łańcuchami dziedziczenia.	
Odin_javascript::10_oop_principles	Na czym polega **Dziedziczenie (Inheritance)** w programowaniu obiektowym i jaką relację opisuje?	Tworzy nową klasę na bazie istniejącej (przejmując jej metody i właściwości). Opisuje relację **"jest czymś" (is-a)**.<br><br>Przykład: `class Dog extends Animal { ... }` (Pies *jest* zwierzęciem). 	
Odin_javascript::10_oop_principles	Na czym polega **Kompozycja (Composition)** w programowaniu obiektowym i jaką relację opisuje?	Polega na budowaniu złożonych obiektów poprzez wstrzykiwanie mniejszych, niezależnych części (komponentów). Opisuje relację **"ma coś / składa się z" (has-a)**.<br><br>Przykład: `class Car { constructor() { this.engine = new Engine()	 } }` (Samochód *ma* silnik). 
Odin_javascript::10_oop_principles	Jaki klasyczny problem z dziedziczeniem jest często określany jako problem "Goryla i Banana" (Gorilla Banana problem)?	Problem polega na tym, że potrzebujesz tylko jednej prostej funkcjonalności (banana), ale zmuszony do dziedziczenia otrzymujesz "goryla trzymającego banana i całą dżunglę" (ogromny, niepotrzebny bagaż stanu i metod z klas bazowych).	
Odin_javascript::10_oop_principles	Dlaczego główna zasada projektowa brzmi: **"Preferuj kompozycję ponad dziedziczenie"** (Favor composition over inheritance)?	Ponieważ kompozycja jest znacznie **bardziej elastyczna**. Pozwala przypisywać i wymieniać zachowania w trakcie działania programu (runtime) oraz pozwala uniknąć sztywnych, kruchych i skomplikowanych łańcuchów hierarchii (compile-time). 	
Odin_javascript::10_oop_principles	Czym jest **Problem Diamentu** (Diamond Problem) związany z dziedziczeniem wielokrotnym?	To konflikt, który pojawia się, gdy klasa `D` dziedziczy po klasach `B` i `C`, a obie te klasy dziedziczą po klasie `A` (i modyfikują jej metodę). Nie wiadomo, którą wersję metody klasa `D` powinna odziedziczyć.<br><br>Kompozycja rozwiązuje ten problem, ponieważ bezpośrednio określamy, jakich komponentów używa obiekt. 	
Odin_javascript::10_oop_principles	Jak zaimplementować kompozycję w JavaScript używając Factory Functions zamiast klas? (Podaj przykład implementacji Kaczki, która umie pływać i latać)	Tworząc osobne obiekty z zachowaniami i łącząc je przy użyciu `Object.assign`.<br><br>Przykład:<br><code>const canFly = () => ({ fly: () => 'Lecę' })	</code><br><code>const canSwim = () => ({ swim: () => 'Pływam' });</code><br><code>const createDuck = () => Object.assign({}, canFly(), canSwim());</code>
Odin_javascript::10_oop_principles	W jakim scenariuszu dziedziczenie nadal ma sens i jest poprawne?	Dziedziczenie jest poprawne, gdy relacja **"is-a" (jest czymś)** jest całkowicie prawdziwa i stabilna w każdym możliwym przypadku, a klasa bazowa nie ma skłonności do częstych zmian (np. `class Button extends HTMLElement`).

Odin_javascript::11_linting	Style guides		
Odin_javascript::11_linting	Code style is important! Having a consistent set of style rules for things such as indentation, preferred quote style or general code structure practices, makes your code more maintainable and easier to read. There are several popular JavaScript style guides on the net that set standards for these types of things and you’ll find they often differ in what they enforce. None are “right” or “wrong”, only that they enforce something to promote consistency across a codebase.		
Odin_javascript::11_linting	Here are a few examples of style guides, including ones used by specific companies:		
Odin_javascript::11_linting	The Airbnb Style Guide is one of the most popular.		
Odin_javascript::11_linting	There is also a JavaScript style guide used at Google.		
Odin_javascript::11_linting	The JavaScript Standard Style.		
Odin_javascript::11_linting	Linting		
Odin_javascript::11_linting	The style guides we mentioned above are full of really helpful advice for formatting, organizing and composing your code. But there are a lot of rules - it can be difficult to internalize them all. Linters are tools that will scan your code with a set of style rules and will report any errors to you that they find. In some cases, they can even auto-fix the errors! There are many linters that exist for JavaScript but by far the most common one is ESLint.		
Odin_javascript::11_linting	ESLint is installed as a dev dependency in your project which will allow you to run checks on any of your files via the command line. ESLint’s official “Getting Started” page is a good place to start which covers installation and basic configuration. The default rule set covers many of the most common scenarios with sensible default settings.		
Odin_javascript::11_linting	You will also want to look at the docs on configuring ESLint for a list of options that you can change, such as including or excluding certain folders or files, and details about specific rules.		
Odin_javascript::11_linting	Formatters		
Odin_javascript::11_linting	Formatters are awesome. They are similar to linters, but serve a slightly different function. Formatters take your JavaScript code and then automatically format it according to a set of rules. Unlike linters, they do not look for style errors, but specifically target the layout of your code, making intelligent decisions about things like spaces, indentation levels and line-breaks.		
Odin_javascript::11_linting	As usual, there are multiple formatters out there. Prettier is a very popular choice that is highly opinionated. Besides a few options, most of its formatting decisions are not customizable. Since many of these decisions have been made for you, this reduces the time spent deciding on things like indentation size or spacing, and more time on the problems that actually matter.		
Odin_javascript::11_linting	Like with ESLint, Prettier is installed as a dev dependency in your project, so read Prettier’s installation guide for instructions on how to do this. While it normally runs with its default rules, you can also change any of its settings in a Prettier configuration file. If you are using the default ESLint recommended rule set, there will not be any clashing lint/format rules, meaning you can just install Prettier as is without configuring eslint-config-prettier.		
Odin_javascript::11_linting	Using Prettier makes coding faster and easier! You don’t have to worry about nailing things like indentation, or remembering every semi-colon because it will take care of those details for you.		
Odin_javascript::11_linting	IDE extensions for linting and formatting		
Odin_javascript::11_linting	Linters and formatters are typically packages you install in a project and use via the command line. However, many popular tools also have IDE extensions, and ESLint and Prettier both have extensions for Visual Studio Code that can make linting and formatting much more convenient on your machine.		
Odin_javascript::11_linting	For example, when installed, the ESLint extension can provide linter warnings and errors as color-coded squiggly lines directly in the open file and even give you details about the specific rule(s) broken, all without you having to run ESLint in the command line. The Prettier extension allows you to format a file with an IDE command or custom keyboard shortcut, again without having to run a command in the terminal.		
Odin_javascript::11_linting	It is important that you still have the packages installed as dependencies in your project along any configuration files. The extensions can have fallback rules set, but if they detect the respective package and configuration file in your project, they will use those rules and the package version installed. That way, projects always hold the source of truth for what linting and formatting rules should be applied, and should you ever work on other projects, you’re less likely to introduce unwanted style changes from your local settings.		
Odin_javascript::11_linting	In summary, the extensions are great tools for convenience but they should not be used as the source of truth for a project’s linting or formatting setup.

Odin_javascript::11_linting	Recall template repositories? You can include linter and formatter setup in any of your templates to make things quicker and easier in the future!

Odin_javascript::12_form_validation_with_javascript	Jak nazywa się interfejs API wbudowany w przeglądarki, który udostępnia metody i właściwości do walidacji formularzy w JavaScript?	**Constraint Validation API** (API walidacji ograniczeń).	
Odin_javascript::12_form_validation_with_javascript	Co zwraca właściwość `element.validity` wywołana na polu formularza?	Zwraca obiekt `ValidityState`, który zawiera szereg właściwości (np. `valid`, `valueMissing`, `tooShort`) opisujących szczegółowy stan poprawności tego pola. 	
Odin_javascript::12_form_validation_with_javascript	Jak w JavaScript sprawdzić, czy pole formularza z atrybutem `required` zostało pozostawione puste, korzystając z obiektu `validity`?	Sprawdzając, czy właściwość `element.validity.valueMissing` zwraca `true`.	
Odin_javascript::12_form_validation_with_javascript	Która właściwość obiektu `ValidityState` zwraca `true`, gdy wartość w polu `<input type="email">` lub `url` nie ma poprawnego formatu?	`element.validity.typeMismatch`	
Odin_javascript::12_form_validation_with_javascript	Jaka metoda w JavaScript pozwala przypisać polu formularza niestandardowy komunikat błędu i zablokować jego wysłanie?	`element.setCustomValidity("Twój komunikat błędu")`. 	
Odin_javascript::12_form_validation_with_javascript	Co należy przekazać do metody `setCustomValidity()`, aby wyczyścić wcześniej ustawiony błąd i uznać pole za poprawne?	Należy przekazać pusty ciąg znaków: `element.setCustomValidity("")`.	
Odin_javascript::12_form_validation_with_javascript	Co się dzieje, gdy dodasz atrybut `novalidate` do tagu `<form>`?	Wyłącza on domyślne dymki błędów przeglądarki (pozwala to na zbudowanie własnego UI dla błędów), ale **nie wyłącza** działania Constraint Validation API ani pseudoklas CSS (takich jak `:invalid` czy `:valid`). 	
Odin_javascript::12_form_validation_with_javascript	Jaka metoda Constraint Validation API sprawdza poprawność pola i automatycznie triggeruje pojawienie się domyślnych dymków z błędami (często używana razem z `preventDefault()` w zdarzeniu submit)?	`element.reportValidity()`	
Odin_javascript::12_form_validation_with_javascript	Jaka jest różnica między właściwościami `patternMismatch` a `typeMismatch` w obiekcie `ValidityState`?	`patternMismatch` sprawdza, czy wartość pasuje do wyrażenia regularnego zdefiniowanego w atrybucie `pattern`, podczas gdy `typeMismatch` sprawdza wbudowaną składnię dla określonych typów inputów (jak `email` czy `url`).	
Odin_javascript::13_ecmascript	Czym jest ECMAScript w stosunku do JavaScriptu?	To oficjalny standard języka (utrzymywany przez organizację Ecma International), do którego stosuje się język JavaScript.	
Odin_javascript::13_ecmascript	Czym jest ES6 (lub ES2015)?	To przełomowa aktualizacja JavaScriptu wydana latem 2015 roku, która wprowadziła do języka ogromną liczbę nowych funkcji i ulepszeń składni.	
Odin_javascript::13_ecmascript	Jak zmienił się harmonogram wydań i nazewnictwo ECMAScript po debiucie ES6?	Komitet TC39 przeszedł z wersji liczbowych (ES6, ES7) na lata wydań (ES2015, ES2016) i rozpoczął publikowanie **corocznych**, mniejszych aktualizacji zamiast rzadkich, gigantycznych wydań. 	
Odin_javascript::13_ecmascript	Jaki jest główny problem z używaniem najnowszych funkcji JavaScriptu w projektach produkcyjnych?	Brak pełnego wsparcia we wszystkich przeglądarkach. Zanim nowa funkcja stanie się "szeroko dostępna" na rynku, mogą minąć lata, co grozi wywaleniem się aplikacji u użytkowników ze starszym oprogramowaniem.	
Odin_javascript::13_ecmascript	Czym jest Babel w ekosystemie JavaScript?	To narzędzie, które **transpiluje** (przekształca) nowoczesny kod JavaScript na starszą składnię (np. ES5), która jest bezpieczna i w pełni zrozumiała dla starszych przeglądarek.	
Odin_javascript::14_asynchronous_code	Dlaczego JavaScript potrzebuje funkcji asynchronicznych (asynchronous functions)?	Ponieważ niektóre operacje (np. pobieranie danych z serwera) zajmują dużo czasu, a funkcje asynchroniczne pozwalają na ich wykonywanie w tle, nie blokując reszty działającego kodu.	
Odin_javascript::14_asynchronous_code	Co to jest **Callback** (funkcja wywołania zwrotnego)?	To funkcja przekazywana do innej funkcji jako argument, która jest następnie wywoływana wewnątrz tej funkcji zewnętrznej w celu wykonania jakiejś akcji (np. funkcja reagująca na kliknięcie w `addEventListener`). 	
Odin_javascript::14_asynchronous_code	Co to jest "Callback hell" (piekło wywołań zwrotnych)?	To sytuacja powstająca przy nadużywaniu callbacków, gdy trzeba łączyć wiele z nich w określonej, zależnej od siebie kolejności. Prowadzi to do głębokiego zagnieżdżenia kodu, który staje się nieczytelny i trudny w utrzymaniu. 	
Odin_javascript::14_asynchronous_code	Czym w uproszczeniu jest **Promise** (Obietnica) w JavaScript?	To obiekt, który reprezentuje wartość, jaka może zostać wyprodukowana (zwrócona) w pewnym momencie w przyszłości (np. odpowiedź z zapytania do API).	
Odin_javascript::14_asynchronous_code	Dlaczego przypisanie wyniku asynchronicznej funkcji bezpośrednio do zmiennej (np. `const data = getData()`) zazwyczaj skutkuje wartością `undefined`?	Ponieważ kod wykonuje się dalej bez czekania. Próba odczytania danych następuje, zanim asynchroniczna funkcja `getData()` zdąży je faktycznie pobrać i zwrócić.	
Odin_javascript::14_asynchronous_code	Do czego służy metoda `.then()` wywoływana na obiekcie Promise?	Zatrzymuje ona wykonanie zawartego w niej kodu do momentu, aż Promise zostanie rozwiązany (resolved). Mówi programowi, aby poczekał na dane, a DOPIERO WTEDY (then) uruchomił na nich przekazaną funkcję.	
Odin_javascript::14_asynchronous_code	Jaka jest składnia konstruktora obiektu Promise?	`new Promise(function(resolve, reject) { /* kod wykonawczy (executor) */ })`	
Odin_javascript::14_asynchronous_code	Kiedy uruchamiana jest funkcja `executor` przekazana do konstruktora `new Promise`?	Automatycznie i natychmiastowo w momencie tworzenia obiektu Promise.	
Odin_javascript::14_asynchronous_code	Jakie dwie **wewnętrzne właściwości** posiada obiekt Promise i jakie są ich wartości początkowe?	`state` (początkowo `"pending"`) oraz `result` (początkowo `undefined`). 	
Odin_javascript::14_asynchronous_code	Jak zmieniają się właściwości `state` i `result` obiektu Promise po wywołaniu `resolve(value)`?	`state` zmienia się na `"fulfilled"`, a `result` zmienia się na przekazane `value`. 	
Odin_javascript::14_asynchronous_code	Jak zmieniają się właściwości `state` i `result` obiektu Promise po wywołaniu `reject(error)`?	`state` zmienia się na `"rejected"`, a `result` zmienia się na przekazany `error`.	
Odin_javascript::14_asynchronous_code	Co w kontekście Promise oznacza termin **"settled"** (rozstrzygnięty)?	Oznacza, że Promise zakończył działanie – nie jest już `"pending"`, lecz został spełniony (`"fulfilled"`) albo odrzucony (`"rejected"`).	
Odin_javascript::14_asynchronous_code	Co się stanie, jeśli funkcja `executor` wywoła `resolve()` lub `reject()` wielokrotnie?	Tylko pierwsze wywołanie zostanie uwzględnione. Stan Promise po rozstrzygnięciu jest ostateczny, a wszelkie kolejne wywołania są ignorowane.	
Odin_javascript::14_asynchronous_code	Jakie argumenty przyjmuje metoda `.then()`?	Przyjmuje dwie funkcje (callbacks): pierwsza uruchamia się przy sukcesie (otrzymuje wynik), a druga przy błędzie (otrzymuje obiekt błędu). Składnia: `.then(onFulfilled, onRejected)`.	
Odin_javascript::14_asynchronous_code	Czym technicznie jest metoda `.catch(f)`?	Jest to dokładny odpowiednik i skrót (syntactic sugar) dla wywołania `.then(null, f)`.	
Odin_javascript::14_asynchronous_code	Jakie są 3 główne cechy handlera w metodzie `.finally(f)`?	1. Nie przyjmuje żadnych argumentów (nie wie, czy zadanie zakończyło się sukcesem, czy błędem).<br>2. "Przepuszcza" (passes through) wynik lub błąd do kolejnego handlera w łańcuchu.<br>3. Zwracane przez niego wartości są ignorowane (wyjątkiem jest wyrzucenie błędu przez `throw`).	
Odin_javascript::14_asynchronous_code	Co się stanie, jeśli przypniesz handlery (`.then`, `.catch`) do obiektu Promise, który już wcześniej został rozstrzygnięty (settled)?	Handlery te zostaną wykonane natychmiastowo z dostępnym już wynikiem.	
Odin_javascript::14_asynchronous_code	Jakie są przewagi używania Promises w porównaniu do tradycyjnych Callbacks?	1. Naturalna kolejność (najpierw wywołanie asynchroniczne, potem określenie co zrobić z wynikiem przez `.then`).<br>2. Możliwość przypięcia wielu niezależnych handlerów do jednego obiektu Promise w dowolnym momencie.<br>3. Brak wymogu podawania funkcji zwrotnej (callback) z góry w momencie wywoływania funkcji asynchronicznej.	
Odin_javascript::15_working_with_apis	Co to jest API (Application Programming Interface) w kontekście web developmentu?	To zestaw reguł i mechanizmów pozwalających na komunikację między aplikacjami. Często odnosi się do serwerów udostępniających surowe dane (np. pogodowe, giełdowe) dla innych stron lub aplikacji. 	
Odin_javascript::15_working_with_apis	Dlaczego większość publicznych API wymaga użycia "API Key" (klucza API)?	Aby śledzić, kto wykonuje zapytania, chronić serwery przed nadużyciami (limitowanie zapytań) oraz umożliwiać monetyzację usługi (płatne plany za większy limit zapytań).	
Odin_javascript::15_working_with_apis	Dlaczego umieszczanie prywatnego (szczególnie płatnego) klucza API bezpośrednio w kodzie frontendowym (JavaScript po stronie przeglądarki) jest niebezpieczne?	Ponieważ kod frontendowy jest publicznie widoczny dla każdego użytkownika. Ktoś może łatwo podejrzeć Twój klucz (np. w narzędziach deweloperskich) i wykorzystać go do własnych celów, za co Ty poniesiesz koszty.	
Odin_javascript::15_working_with_apis	Jaka jest obecnie nowoczesna, natywna funkcja przeglądarki używana do pobierania danych z API (zastępująca starsze `XMLHttpRequest`)?	Funkcja `fetch()`.	
Odin_javascript::15_working_with_apis	Co dokładnie zwraca wywołanie funkcji `fetch('url')` w JavaScript?	Zwraca obiekt **Promise**, który po rozwiązaniu (resolve) dostarcza obiekt `Response` (odpowiedź z serwera). 	
Odin_javascript::15_working_with_apis	Gdy otrzymasz obiekt `Response` z funkcji `fetch()`, jak przekształcić jego ciało (body) na format JSON (czytelny obiekt JavaScript)?	Należy wywołać metodę `.json()` na obiekcie odpowiedzi (np. `response.json()`), co ponownie zwróci **Promise**, który rozwiąże się do sparsowanych danych.	
Odin_javascript::15_working_with_apis	Jak prawidłowo obsłużyć błędy sieciowe (np. brak połączenia z internetem) w łańcuchu wywołań `fetch()`?	Używając metody `.catch()` na końcu łańcucha obietnic (np. `.catch(error => console.error(error))`).	
Odin_javascript::15_working_with_apis	Czy wywołanie `fetch()` zostanie odrzucone (wejdzie do bloku `.catch()`), jeśli serwer zwróci błąd HTTP taki jak `404 Not Found` lub `500 Internal Server Error`?	**Nie**. `fetch()` odrzuca Promise tylko w przypadku awarii sieci (np. brak internetu). Błędy HTTP są traktowane jako udane zapytania (serwer przecież odpowiedział), więc trzeba ręcznie sprawdzać właściwość `response.ok` lub `response.status` w bloku `.then()`.	
Odin_javascript::17_testing_basics	Co oznacza skrót **TDD** i na czym polega ta technika programowania?	**Test Driven Development**. Polega na pisaniu zautomatyzowanych testów **przed** napisaniem właściwego kodu aplikacji, który ma być testowany. 	
Odin_javascript::17_testing_basics	Jakie są trzy podstawowe kroki (fazy) w cyklu TDD?	1. **Red** (Czerwony) - napisz prosty, automatyczny test, który na pewno nie przejdzie (bo kod jeszcze nie istnieje).<br>2. **Green** (Zielony) - napisz jak najszybciej i najprościej kod (nawet brzydki), byle tylko test przeszedł.<br>3. **Refactor** (Refaktoryzacja) - zoptymalizuj kod i usuń zduplikowane fragmenty, zachowując działające testy.	
Odin_javascript::17_testing_basics	Dlaczego faza "Green" w TDD, polegająca na pisaniu najprostszego możliwego kodu, bywa trudna dla doświadczonych programistów? Odin_javascript::17_testing_basics	Jaką zaletę ma podejście "napisz brzydki kod, byle przeszedł test", a następnie refaktoryzuj (w porównaniu do pisania od razu docelowego, skomplikowanego kodu)?	Gwarantuje, że nowy kod na pewno działa zgodnie z założeniami od samego początku, a proces jego udoskonalania jest chroniony przez istniejące, "zielone" testy, co zmniejsza ryzyko wprowadzenia bugów.	
Odin_javascript::17_testing_basics	Czym różni się TDD od "testów jednostkowych" (Unit Tests)?	Testy jednostkowe to *rodzaj* testów (sprawdzających pojedyncze funkcje/moduły). TDD to *metodologia* i *technika programowania*, w której pisanie testów steruje procesem tworzenia aplikacji.	
Odin_javascript::17_testing_basics	Dlaczego mówi się, że "TDD nie polega tak naprawdę na testowaniu" (lub co jest ważniejsze: składnia czy filozofia)?	TDD to przede wszystkim technika *myślenia* o kodzie i jego projektowaniu. Filozofia (dlaczego i co testować) jest kluczowa, a same testy (i konkretna składnia, np. Jest) to tylko wtórny, użyteczny skutek tego procesu.	
Odin_javascript::17_testing_basics	Jakie obawy (i "wymówki") często powstrzymują programistów przed stosowaniem Test Driven Development (TDD)?	TDD często wydaje się żmudne ("nie mam czasu na testy") i postrzegane jest jako opcjonalny, zbędny dodatek (tzw. "gold plating"), który spowalnia kreatywność i dostarczanie projektu na czas.	
Odin_javascript::17_testing_basics	Jakie są popularne biblioteki do testowania w JavaScript i czy trudno się między nimi przesiadać?	Do najpopularniejszych należą Jest, Mocha, Jasmine i Tape. Przesiadka jest stosunkowo łatwa, ponieważ ich podstawowa składnia jest niemal identyczna.	
Odin_javascript::17_testing_basics	Jaki system modułów jest domyślnie wspierany przez środowisko uruchomieniowe testów Jest?	**CommonJS** (CJS), wykorzystujący instrukcje `require` oraz `module.exports`.	
Odin_javascript::17_testing_basics	Co jest potrzebne (jaki pakiet i plik), aby móc używać nowoczesnej składni modułów ES6 (ESM - `import`/`export`) w testach pisanych w Jest?	Należy zainstalować pakiet `@babel/preset-env` (jako `devDependency`) oraz utworzyć plik konfiguracyjny `babel.config.js`. 	
Odin_javascript::17_testing_basics	W jaki sposób Babel sprawia, że Jest "rozumie" moduły ES6 (ESM)? Odin_javascript::17_testing_basics	Czym są "Matchers" (Dopasowania) w bibliotece Jest?	To zbiór przydatnych funkcji (używanych np. po wywołaniu `expect()`), które służą do sprawdzania, czy otrzymana wartość spełnia określone warunki i czy zgadza się z wartością oczekiwaną.	
Odin_javascript::18_more_testing	Dlaczego podczas testowania należy dbać o izolację (testowanie jednej metody na raz)?	Aby w przypadku niepowodzenia testu móc jak najszybciej i najdokładniej zlokalizować przyczynę błędu, bez obawy, że zawiodła jakaś zewnętrzna funkcja.	
Odin_javascript::18_more_testing	Czym jest kod "ściśle powiązany" (tightly coupled) w kontekście testowania?	To kod, w którym funkcje bezpośrednio wywołują inne funkcje, manipulują DOM lub polegają na zewnętrznym stanie, co sprawia, że są bardzo trudne do przetestowania w izolacji. 	
Odin_javascript::18_more_testing	Jakie dwa warunki musi spełniać "Czysta Funkcja" (Pure Function)?	1. Dla tych samych argumentów wejściowych zawsze zwraca ten sam wynik.<br>2. Nie wywołuje żadnych **efektów ubocznych** (side effects). 	
Odin_javascript::18_more_testing	Czym są efekty uboczne (side effects) w programowaniu i dlaczego utrudniają testowanie?	To wszelkie interakcje funkcji ze światem zewnętrznym, np. modyfikacja DOM, zapytania do bazy danych, zmiana zmiennych globalnych czy nawet `console.log`. Utrudniają testowanie, bo wymagają symulowania całego środowiska wokół funkcji.	
Odin_javascript::18_more_testing	Co należy zrobić w pierwszej kolejności, zanim spróbuje się przetestować ściśle powiązany (tightly coupled) kod?	Należy spróbować go rozdzielić (decouple) – np. wydzielić czystą logikę obliczeniową do osobnych, łatwych do przetestowania funkcji, oddzielając ją od interakcji z DOM czy API.	
Odin_javascript::18_more_testing	Czym jest Mockowanie (Mocking) w testowaniu oprogramowania?	To tworzenie "fałszywych" wersji funkcji, obiektów lub modułów, które zachowują się w kontrolowany, z góry określony sposób (np. zawsze zwracają konkretną wartość). 	
Odin_javascript::18_more_testing	Kiedy używa się mocków (mock functions) w testach?	Gdy chcemy przetestować funkcję w izolacji, ale zależy ona od zewnętrznych, niemożliwych do łatwego odłączenia czynników (np. pobieranie danych z inputu DOM, zewnętrzne API, odczyt z pliku).	
Odin_javascript::18_more_testing	Jak powinno się testować "przychodzące zapytania" (incoming query messages)?	Poprzez asercję ich **wyniku** – sprawdzając, czy funkcja zwraca oczekiwaną wartość na podstawie przekazanych argumentów.	
Odin_javascript::18_more_testing	Dlaczego **nie** powinno się testować szczegółów implementacji (implementation details)?	Sprawia to, że testy stają się bardzo kruche (brittle). Każda zmiana w tym, *jak* funkcja działa pod spodem (refaktoryzacja), popsuje test, nawet jeśli końcowy wynik pozostanie poprawny. Należy testować to, *co* funkcja robi (interfejs), a nie *jak* to robi.	
Odin_javascript::18_more_testing	Czy powinno się pisać testy bezpośrednio dla metod prywatnych (private methods)?	Nie. Metody prywatne to szczegół implementacji. Powinny być testowane pośrednio, poprzez testowanie publicznego API (publicznych metod), które z nich korzysta wewnątrz klasy.	
Odin_javascript::18_more_testing	Dlaczego nie powinno się testować wychodzących zapytań (zależności), które nie wywołują efektów ubocznych (outgoing queries with no side effects)?	Ponieważ testowanie, czy po prostu "zapytaliśmy" inny obiekt o dane, wiąże test ze szczegółami implementacji. Należy testować tylko wychodzące **komendy** (które zmieniają stan systemu), upewniając się za pomocą mocków, że zostały wywołane z odpowiednimi argumentami.

Odin_javascript::25_a_deeper_look_at_git	W jaki sposób Git wie, jakie commity należą do danej gałęzi, skoro gałąź wskazuje tylko na *jeden* commit?	Ponieważ każdy commit w Gicie również działa jak wskaźnik – posiada on referencję (wskaźnik) do swojego **rodzica** (poprzedniego commita). Odin_javascript::25_a_deeper_look_at_git	Do czego służy specjalny wskaźnik `HEAD` w Gicie?	Wskazuje on na aktualnie wybraną ("odcheckoutowaną") gałąź lub bezpośrednio na konkretny commit, określając, w jakim miejscu historii znajduje się Twój obecny katalog roboczy (working directory).

Odin_javascript::25_a_deeper_look_at_git	Jakie niebezpieczeństwo wiąże się z użyciem komendy `git reset --hard`?	Jest to operacja destrukcyjna. Odin_javascript::5_es6_modules	Jak wyeksportować wiele zmiennych jako **Eksporty Nazwane** na końcu pliku?	`export { var1, var2 }	` (Uwaga: to *nie jest* literał obiektu).
Odin_javascript::5_es6_modules	Jak zaimportować **Eksporty Nazwane**?	Używając nawiasów klamrowych: `import { var1, var2 } from './file.js'	`
Odin_javascript::5_es6_modules	Ile **Eksportów Domyślnych** może mieć jeden plik?	Tylko **jeden**.	
Odin_javascript::5_es6_modules	Jak wyeksportować wartość jako **Eksport Domyślny**?	`export default myValue	` LUB `export default variableName;`
Odin_javascript::5_es6_modules	Jak zaimportować **Eksport Domyślny**?	Bez nawiasów klamrowych, używając dowolnej wybranej nazwy: `import myName from './file.js'	`
Odin_javascript::5_es6_modules	Czy można łączyć eksporty **Nazwane** i **Domyślne** w tym samym pliku?	Tak.	
Odin_javascript::5_es6_modules	Jak zaimportować jednocześnie eksport domyślny i nazwany z tego samego pliku?	`import defaultName, { named1, named2 } from './path.js'	`
Odin_javascript::5_es6_modules	Czy ścieżka pliku w instrukcji `import` obsługuje template literals (np. `` `./${file}.js` ``)?	Nie, musi to być statyczny ciąg znaków (w cudzysłowie pojedynczym lub podwójnym).	
Odin_javascript::5_es6_modules	Jaki atrybut należy dodać do znacznika `<script>`, aby włączyć obsługę modułów ES6 w przeglądarce?	`type="module"` (np. `<script src="main.js" type="module"></script>`).	
Odin_javascript::5_es6_modules	Co to jest **CommonJS** (CJS)?	To system modułów tradycyjnie używany w **Node.js** (korzystający z `require` i `module.exports`).	
Odin_javascript::5_es6_modules	Czy przeglądarki natywnie rozumieją składnię CommonJS (`require`)?	Nie, przeglądarki natywnie wspierają **ESM** (`import`/`export`).	
Odin_javascript::6_npm	Co to jest **npm**?	To menedżer pakietów (package manager) dla JavaScript – ogromne repozytorium narzędzi i bibliotek oraz program wiersza poleceń do zarządzania nimi.	
Odin_javascript::6_npm	Czy skrót **npm** oficjalnie oznacza "Node Package Manager"?	Nie (choć często jest tak błędnie nazywany, oficjalnie to po prostu "npm").	
Odin_javascript::6_npm	Jaki plik jest "sercem" każdego projektu zarządzanego przez npm?	`package.json` 	
Odin_javascript::6_npm	Do czego służy plik `package.json`?	Jest to plik w formacie JSON zawierający metadane projektu (nazwa, wersja) oraz listę jego **zależności** (bibliotek) i skryptów.	
Odin_javascript::6_npm	Jaka komenda pozwala automatycznie pobrać i zainstalować wszystkie biblioteki wymienione w `package.json`?	`npm install` 	
Odin_javascript::6_npm	Czym są **zależności** (dependencies) w projekcie?	To zewnętrzne pakiety/kody (napisane przez innych), których Twój projekt potrzebuje do poprawnego działania, a które są pobierane z npm.	
Odin_javascript::6_npm	Czym różnią się `dependencies` od `devDependencies`?	`dependencies` to biblioteki niezbędne do działania gotowej aplikacji, a `devDependencies` to narzędzia potrzebne tylko podczas **tworzenia** (np. testy, lintery), które nie trafiają na produkcję.	
Odin_javascript::6_npm	Dlaczego repozytoria kodu (np. na GitHubie) zazwyczaj nie zawierają kodu samych bibliotek (folderu `node_modules`)?	Ponieważ folder ten jest bardzo duży, a każdy programista może go łatwo odtworzyć (pobrać), uruchamiając `npm install` na podstawie pliku `package.json`.	
Odin_javascript::6_npm	Do czego służy sekcja `scripts` w pliku `package.json`?	Pozwala definiować skróty do komend (np. uruchamiania testów czy lintera), które można wywołać za pomocą `npm run <nazwa-skryptu>`.	
Odin_javascript::7_webpack	Co to jest "Module Bundler" (np. Webpack)?	Narzędzie, które bierze plik wejściowy (entry point), analizuje jego zależności (buduje graf zależności) i łączy wiele plików (JS, CSS, obrazy) w jeden lub kilka plików wynikowych gotowych do wysłania do przeglądarki. 	
Odin_javascript::7_webpack	Jaka jest różnica między folderem `src` a `dist` w konwencji Webpacka?	`src` (source) zawiera kod źródłowy, który piszesz. `dist` (distribution) zawiera zoptymalizowany, wynikowy kod wygenerowany przez Webpacka, gotowy do wdrożenia (produkcji).	
Odin_javascript::7_webpack	Jaki jest domyślny plik konfiguracyjny Webpacka?	`webpack.config.js`	
Odin_javascript::7_webpack	Co określa właściwość `entry` w konfiguracji Webpacka?	Ścieżkę do pliku startowego (np. `./src/index.js`), od którego Webpack zaczyna budowanie grafu zależności.	
Odin_javascript::7_webpack	Co określa właściwość `output` w konfiguracji Webpacka?	Obiekt definiujący, gdzie (`path`) i pod jaką nazwą (`filename`) ma zostać zapisany zbudowany plik (bundle).	
Odin_javascript::7_webpack	Co robi opcja `clean: true` w konfiguracji `output`?	Automatycznie czyści folder docelowy (`dist`) przed każdym nowym budowaniem, usuwając stare pliki.	
Odin_javascript::7_webpack	Do czego służy `HtmlWebpackPlugin`?	Automatycznie generuje plik `index.html` w folderze `dist` i automatycznie wstrzykuje do niego tag `<script>` z linkiem do zbudowanego pliku JS.	
Odin_javascript::7_webpack	Jakie dwa loadery są potrzebne do obsługi CSS w Webpacku i co robią?	`css-loader` (zamienia CSS w string JS i obsługuje importy) oraz `style-loader` (wstrzykuje ten CSS do tagu `<style>` w DOM).	
Odin_javascript::7_webpack	W jakiej kolejności Webpack przetwarza loadery zdefiniowane w tablicy `use` (np. `["style-loader", "css-loader"]`)?	**Od końca do początku** (od prawej do lewej). Najpierw uruchomi się `css-loader`, a wynik przekaże do `style-loader`. 	
Odin_javascript::7_webpack	Jak w Webpacku 5 obsługuje się obrazy importowane w JS (bez instalowania dodatkowych loaderów)?	Używając **Asset Modules** poprzez ustawienie `type: "asset/resource"` w regułach (`rules`) dla plików graficznych.	
Odin_javascript::7_webpack	Co należy zrobić, aby obrazek zaimportowany w pliku JS (np. `import img from './pic.png'`) został poprawnie dołączony do bundle'a?	Należy użyć zmiennej z importu jako źródła (np. `image.src = img`), ponieważ Webpack zamieni ten import na ostateczną ścieżkę do pliku w folderze `dist`.	
Odin_javascript::7_webpack	Co to są **Source Maps** (np. `devtool: "eval-source-map"`)?	To mapy, które pozwalają przeglądarce powiązać zminifikowany kod w `dist` z oryginalnym kodem w `src`. Dzięki temu błędy w konsoli wskazują na linię w Twoim kodzie, a nie w bundle'u. 	
Odin_javascript::7_webpack	Do czego służy `webpack-dev-server`?	Uruchamia lokalny serwer, który nasłuchuje zmian w plikach i automatycznie odświeża stronę (Live Reloading), budując aplikację w pamięci RAM (nie zapisuje plików do dysku/dist).	
Odin_javascript::8_revisiting_webpack	Gdzie definiuje się skrypty npm (npm scripts)?	W pliku package.json w obiekcie "scripts".	
Odin_javascript::8_revisiting_webpack	Jak w terminalu uruchomić skrypt npm o nazwie "build" (np. "build": "webpack")?	npm run build	
Odin_javascript::8_revisiting_webpack	Dlaczego wewnątrz skryptów npm nie musimy używać przedrostka npx (np. piszemy "dev": "webpack serve", a nie "dev": "npx webpack serve")?	Ponieważ skrypty npm automatycznie wiedzą, jak korzystać z poleceń dostarczanych przez pakiety zainstalowane lokalnie w projekcie (w node_modules).	
Odin_javascript::8_revisiting_webpack	Jakie są dwa główne tryby (modes) budowania w Webpacku?	development (zoptymalizowany pod kątem pracy lokalnej i debugowania) oraz production (kod jest zminifikowany i zoptymalizowany pod kątem wydajności).	
Odin_javascript::8_revisiting_webpack	Jak za pomocą skryptu npm wymusić na Webpacku użycie konkretnego, niestandardowego pliku konfiguracyjnego (np. webpack.prod.js)?	Dodając flagę --config, np. "build": "webpack --config webpack.prod.js".	
Odin_javascript::8_revisiting_webpack	Do czego służy pakiet webpack-merge?	Pozwala na podzielenie konfiguracji Webpacka na mniejsze, reużywalne pliki (np. wspólna baza, plik dla środowiska dev i plik dla prod) i łączenie ich bez powielania kodu.	
Odin_javascript::8_revisiting_webpack	Czym jest "Template repository" (repozytorium szablonowe) na GitHubie i do czego się przydaje?	To funkcja pozwalająca oznaczyć istniejące repozytorium (np. z gotową konfiguracją Webpacka) jako szablon. Pozwala to na tworzenie nowych projektów będących od razu kopią tego szablonu, co oszczędza czas na powtarzalną konfigurację.	
Odin_javascript::8_revisiting_webpack	Jak uruchomić skrypt npm o nazwie `build` (zdefiniowany w sekcji `scripts` w `package.json`)?	`npm run build`	
Odin_javascript::8_revisiting_webpack	Jakie polecenia npm pozwalają na uruchomienie skryptów `start` i `test` bez użycia słowa `run`?	`npm start` oraz `npm test` (lub `npm t`).	
Odin_javascript::8_revisiting_webpack	W jaki sposób uzyskać dostęp do wersji pakietu określonej w `package.json` wewnątrz skryptu npm?	Używając zmiennej środowiskowej `$npm_package_version`.	
Odin_javascript::8_revisiting_webpack	Jakiej flagi należy użyć, aby skrypt npm nie zakończył się błędem (kodem innym niż 0), jeśli dany skrypt nie istnieje w `package.json`?	`--if-present` (np. `npm run build --if-present`).	
Odin_javascript::8_revisiting_webpack	Jak w skrypcie npm przekazać dodatkowe argumenty do już istniejącego polecenia (np. dodać flagę `--fix` do wywołania skryptu `lint`)?	Używając `--` przed dodatkowymi argumentami, np. `"lint:fix": "npm run lint -- --fix"`.	
Odin_javascript::8_revisiting_webpack	Jak w skrypcie npm (opartym na powłoce unixowej) uruchomić dwa polecenia **sekwencyjnie** (jedno po drugim, tylko jeśli pierwsze się powiedzie)?	Używając operatora `&&` (np. `"build-and-test": "npm run build && npm run test"`).	
Odin_javascript::8_revisiting_webpack	Jak w skrypcie npm uruchomić dwa polecenia **równolegle** (jednocześnie)?	Używając operatora `&` (np. `"build-and-test-parallel": "npm run build & npm run test"`).	
Odin_javascript::8_revisiting_webpack	Do czego służy operator `||` w skryptach npm (powłoce)?	Wykonuje drugie polecenie **tylko wtedy**, gdy pierwsze zakończy się błędem (zwróci kod błędu).	
Odin_javascript::8_revisiting_webpack	Czym są skrypty cyklu życia (lifecycle scripts) w npm (np. `preinstall`, `postinstall`)?	Są to predefiniowane nazwy skryptów, które npm wykonuje automatycznie na określonych etapach działania (np. przed/po instalacji pakietu, budowaniu czy publikacji). 	
Odin_javascript::8_revisiting_webpack	Jak dodać skrypt, który uruchomi się automatycznie **przed** skryptem o nazwie `test`?	Należy utworzyć skrypt o nazwie `pretest`.	
Odin_javascript::8_revisiting_webpack	Dlaczego należy unikać komend takich jak `rm -rf` w skryptach npm i co stosować w zamian?	Ponieważ `rm -rf` zadziała na systemach Linux/Mac, ale spowoduje błąd w systemie Windows. Zamiast tego należy używać narzędzi wieloplatformowych (cross-platform), takich jak pakiet `rimraf`.	
Odin_javascript::8_revisiting_webpack	Jaki kod wyjścia (exit code) powinien zwrócić skrypt, aby zasygnalizować sukces, a jaki, aby zasygnalizować błąd?	`0` dla sukcesu, a kod niezerowy (np. `1`) dla błędu.	
Odin_javascript::9_json	Co to jest JSON i do czego głównie służy?	**J**ava**S**cript **O**bject **N**otation. To lekki, oparty na tekście format zapisu danych, używany powszechnie do przesyłania ustrukturyzowanych danych przez sieć (np. między serwerem a klientem).	
Odin_javascript::9_json	Czym różni się zapis kluczy i tekstów w formacie JSON od standardowych obiektów w JavaScript?	W JSON **wszystkie klucze oraz wartości tekstowe muszą być otoczone podwójnymi cudzysłowami** (`"klucz": "wartość"`). Użycie pojedynczych cudzysłowów jest błędem. 	
Odin_javascript::9_json	Jak nazywa się proces zamiany natywnego obiektu (np. JS) na ciąg znaków JSON, a jak proces odwrotny?	Zamiana obiektu na ciąg znaków (string) to **serializacja** (serialization). Zamiana ciągu znaków z powrotem na obiekt to **deserializacja** (deserialization).	
Odin_javascript::9_json	Jaki jest standardowy typ MIME oraz rozszerzenie dla plików JSON?	Typ MIME to `application/json`, a rozszerzenie pliku to `.json`.	
Odin_javascript::9_json	Jakich typów danych z JavaScript **nie można** przechowywać w formacie JSON?	Nie można przechowywać funkcji (metod), `undefined`, `NaN`, `Infinity` oraz specyficznych obiektów takich jak `Date`, `Map`, `Set`. 	
Odin_javascript::9_json	Czy w pliku JSON dozwolone są komentarze lub przecinki na końcu listy elementów (tzw. trailing commas)?	Nie, w JSON komentarze są zabronione, a zostawienie przecinka na końcu (po ostatnim elemencie obiektu lub tablicy) powoduje błąd składni.	
Odin_javascript::9_json	Jaka wbudowana metoda JavaScript służy do konwersji tekstowego ciągu JSON (string) na obiekt JavaScript?	`JSON.parse()`	
Odin_javascript::9_json	Jaka wbudowana metoda JavaScript służy do konwersji (serializacji) obiektu JavaScript na ciąg znaków JSON?	`JSON.stringify()` 	
Odin_javascript::9_json	Jeśli używasz Fetch API, jaka metoda obiektu `Response` pozwala automatycznie odczytać i sparsować odpowiedź do obiektu JavaScript?	`response.json()` (zwraca Promise, który rozwiązuje się do obiektu JS).	
Odin_javascript::9_json	Co się stanie, jeśli do `JSON.parse()` przekażesz string zawierający prawidłowy obiekt JavaScript, ale w którym liczby są zapisane w systemie szesnastkowym (hex) lub brakuje cudzysłowów wokół kluczy?	Wyrzuci błąd. JSON przyjmuje tylko liczby w formacie dziesiętnym, a każdy klucz bezwzględnie wymaga podwójnych cudzysłowów.
