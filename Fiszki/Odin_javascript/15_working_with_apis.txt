Co to jest API (Application Programming Interface) w kontekście web developmentu?;To zestaw reguł i mechanizmów pozwalających na komunikację między aplikacjami. Często odnosi się do serwerów udostępniających surowe dane (np. pogodowe, giełdowe) dla innych stron lub aplikacji. 
Dlaczego większość publicznych API wymaga użycia "API Key" (klucza API)?;Aby śledzić, kto wykonuje zapytania, chronić serwery przed nadużyciami (limitowanie zapytań) oraz umożliwiać monetyzację usługi (płatne plany za większy limit zapytań).
Dlaczego umieszczanie prywatnego (szczególnie płatnego) klucza API bezpośrednio w kodzie frontendowym (JavaScript po stronie przeglądarki) jest niebezpieczne?;Ponieważ kod frontendowy jest publicznie widoczny dla każdego użytkownika. Ktoś może łatwo podejrzeć Twój klucz (np. w narzędziach deweloperskich) i wykorzystać go do własnych celów, za co Ty poniesiesz koszty.
Jaka jest obecnie nowoczesna, natywna funkcja przeglądarki używana do pobierania danych z API (zastępująca starsze `XMLHttpRequest`)?;Funkcja `fetch()`.
Co dokładnie zwraca wywołanie funkcji `fetch('url')` w JavaScript?;Zwraca obiekt **Promise**, który po rozwiązaniu (resolve) dostarcza obiekt `Response` (odpowiedź z serwera). 
Gdy otrzymasz obiekt `Response` z funkcji `fetch()`, jak przekształcić jego ciało (body) na format JSON (czytelny obiekt JavaScript)?;Należy wywołać metodę `.json()` na obiekcie odpowiedzi (np. `response.json()`), co ponownie zwróci **Promise**, który rozwiąże się do sparsowanych danych.
Jak prawidłowo obsłużyć błędy sieciowe (np. brak połączenia z internetem) w łańcuchu wywołań `fetch()`?;Używając metody `.catch()` na końcu łańcucha obietnic (np. `.catch(error => console.error(error))`).
Czy wywołanie `fetch()` zostanie odrzucone (wejdzie do bloku `.catch()`), jeśli serwer zwróci błąd HTTP taki jak `404 Not Found` lub `500 Internal Server Error`?;**Nie**. `fetch()` odrzuca Promise tylko w przypadku awarii sieci (np. brak internetu). Błędy HTTP są traktowane jako udane zapytania (serwer przecież odpowiedział), więc trzeba ręcznie sprawdzać właściwość `response.ok` lub `response.status` w bloku `.then()`.
