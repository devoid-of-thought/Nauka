Jak zapisać podstawowe zapytanie medialne (media query), które ustawi margines elementu `body` na 8px tylko dla ekranów o szerokości 600px i mniejszych?;`@media (max-width: 600px) { body { margin: 8px; } }`
Jaka jest różnica w działaniu między `max-width` a `min-width` w zapytaniu medialnym (media query)?;`max-width: X` aplikuje style dla ekranów o szerokości **od 0 do X** (włącznie, czyli dla ekranów mniejszych lub równych). `min-width: X` aplikuje style dla ekranów o szerokości **od X w górę** (dla ekranów większych lub równych).
Czym jest tzw. "Breakpoint" w kontekście Responsive Web Design?;Jest to określony, krytyczny punkt (wymiar ekranu, np. 600px), przy którym aktywuje się zapytanie medialne (`@media`) w celu zmiany układu lub stylów strony, ponieważ na poprzednim układzie treść przestała wyglądać dobrze. 
Dlaczego dobrą praktyką jest ograniczanie liczby zapytań medialnych (media queries) do absolutnego minimum?;Zbyt wiele zapytań (np. pod każdy konkretny model telefonu) drastycznie komplikuje kod i utrudnia jego utrzymanie. Należy przede wszystkim polegać na **naturalnej elastyczności układu** (Flexbox/Grid/procenty), a breakpointów używać tylko tam, gdzie układ faktycznie tego wymaga.
Jak przybliżanie strony (zooming) w przeglądarce desktopowej wpływa na działanie zapytań medialnych?;Przybliżenie (zoom-in) zmniejsza *efektywną* rozdzielczość okna przeglądarki. System traktuje to tak, jakby ekran stawał się fizycznie węższy, co powoduje uruchamianie zapytań medialnych przeznaczonych dla mniejszych ekranów (np. mobilnych).
Jakiego zapytania medialnego użyjesz, aby zmienić kolory na czarno-białe i ukryć przyciski nawigacyjne w momencie, gdy użytkownik będzie chciał **wydrukować** Twoją stronę internetową?;Należy użyć zapytania z typem nośnika `print`: `@media print { /* style do druku, np. nav { display: none; } */ }`
Jaka jest fundamentalna różnica między zapytaniem medialnym (`@media`) a zapytaniem kontenera (`@container`)?;Zapytanie medialne (`@media`) reaguje na zmiany wymiarów **całego okna przeglądarki (viewportu)**. Zapytanie kontenera (`@container`) reaguje na zmiany wymiarów **bezpośredniego kontenera (rodzica)** danego elementu, niezależnie od tego, jak duże jest w tym momencie okno przeglądarki.
